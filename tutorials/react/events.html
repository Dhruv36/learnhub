Events & Interactivity Foundations

What you'll master: React's synthetic event system and how it differs from raw DOM events (delegation at the root, pooling history, onChange's real meaning), handler patterns — passing arguments, propagation control, default prevention — plus keyboard and pointer events done accessibly, and when to drop down to native listeners. Prerequisites: State and the JS Events lesson — bubbling, capture, and delegation transfer directly.

Part 1 — Synthetic Events: What React Actually Does

You never call addEventListener per element. React attaches its listeners once at the root container and dispatches to your handlers itself — delegation as a framework service (the pattern you built by hand in JS Events):

<button onClick={handleClick}>Save</button>
// What React does:
// 1. The root container has one real "click" listener (React 17+: the root, not document).
// 2. A click bubbles up to it; React looks up which fiber the target belongs to.
// 3. React builds a SyntheticEvent — a cross-browser wrapper with the same interface
//    (e.target, e.preventDefault(), e.stopPropagation()…) around the native event.
// 4. It "bubbles" the synthetic event along the FIBER tree, calling your onClickCapture
//    handlers down, then onClick handlers up.
//
// Practical consequences:
// • e.nativeEvent gets you the raw event if you need it.
// • React's bubbling follows the COMPONENT tree — even across portals (a modal
//   rendered into document.body still bubbles clicks to its React parent!).
// • stopPropagation() on a synthetic event won't stop a NATIVE listener you
//   attached to document — different dispatch systems.
Legacy interview flag: "event pooling" (reusing event objects, the old e.persist() dance) was removed in React 17. If asked, say so — knowing it's gone signals current knowledge; explaining how to work around it signals 2019.

Part 2 — Handler Patterns

function TodoRow({ todo, onToggle, onDelete }) {
  // 1. Passing arguments — wrap in an arrow (this is the idiom, not a hack):
  return (
    <li>
      <input type="checkbox" checked={todo.done}
             onChange={() => onToggle(todo.id)} />
      {todo.text}
      <button onClick={() => onDelete(todo.id)}>✕</button>
    </li>
  );
}

// 2. The event object is the handler's argument:
function SearchBox({ onSearch }) {
  return <input onChange={e => onSearch(e.target.value)} />;   // e.target = the DOM node
}

// 3. One handler, many controls — dispatch on name/dataset (delegation in miniature):
function Toolbar({ onAction }) {
  const handle = e => onAction(e.target.closest("button")?.dataset.action);
  return (
    <div onClick={handle}>                      {/* one handler on the container */}
      <button data-action="bold">B</button>
      <button data-action="italic">I</button>
      <button data-action="link">🔗</button>
    </div>
  );
}
// Note: within React you usually DON'T need manual delegation for performance —
// React already delegates. Use it when it simplifies code, not to "save listeners".

Part 3 — preventDefault & stopPropagation, Precisely

// preventDefault: cancel the BROWSER's built-in action, keep bubbling:
function LinkTab({ href, onSelect }) {
  return <a href={href} onClick={e => { e.preventDefault(); onSelect(href); }}>…</a>;
  // keeps SPA navigation client-side; href stays for middle-click/SEO/a11y
}

<form onSubmit={e => { e.preventDefault(); save(); }}>   // stop full-page reload — THE form idiom

// stopPropagation: stop PARENT handlers from seeing it, keep default action:
function Card({ onOpen, onFavorite }) {
  return (
    <article onClick={onOpen}>                     {/* whole card is clickable */}
      <h3>Title</h3>
      <button onClick={e => { e.stopPropagation(); onFavorite(); }}>⭐</button>
      {/* without it: favoriting ALSO opens the card */}
    </article>
  );
}

Part 4 — Build It: an Accessible Dropdown Menu

The component that exercises every event skill at once — clicks, outside-clicks, keyboard, focus:

function Dropdown({ label, options, onPick }) {
  const [open, setOpen] = useState(false);
  const [active, setActive] = useState(0);          // highlighted option (keyboard)
  const rootRef = useRef(null);

  // Outside-click: a NATIVE listener on document — React events can't see
  // clicks on other components; this is the sanctioned escape hatch:
  useEffect(() => {
    if (!open) return;
    const onDocClick = e => {
      if (!rootRef.current.contains(e.target)) setOpen(false);
    };
    document.addEventListener("pointerdown", onDocClick);
    return () => document.removeEventListener("pointerdown", onDocClick);   // ALWAYS cleanup
  }, [open]);

  function onKeyDown(e) {
    switch (e.key) {                                // e.key, never deprecated e.keyCode
      case "ArrowDown": e.preventDefault(); setActive(a => Math.min(a + 1, options.length - 1)); break;
      case "ArrowUp":   e.preventDefault(); setActive(a => Math.max(a - 1, 0)); break;
      case "Enter":
      case " ":         e.preventDefault(); pick(options[active]); break;
      case "Escape":    setOpen(false); break;
    }
  }
  function pick(opt) { onPick(opt); setOpen(false); }

  return (
    <div ref={rootRef} onKeyDown={onKeyDown}>
      <button aria-haspopup="listbox" aria-expanded={open}
              onClick={() => setOpen(o => !o)}>{label} ▾</button>
      {open && (
        <ul role="listbox">
          {options.map((opt, i) => (
            <li key={opt} role="option" aria-selected={i === active}
                className={i === active ? "active" : ""}
                onPointerEnter={() => setActive(i)}
                onClick={() => pick(opt)}>{opt}</li>
          ))}
        </ul>
      )}
    </div>
  );
}
// The takeaways in miniature: React events for everything inside the component;
// one native document listener (with cleanup) for outside-clicks; keyboard parity
// with pointer interactions; ARIA state mirrors component state.

Interview Questions

Beginner: What is a SyntheticEvent and why does React use one?

A cross-browser wrapper around the native event with the standard interface (target, preventDefault, stopPropagation). React attaches real listeners once at the root, then constructs synthetic events and dispatches them through the component tree — giving consistent behavior across browsers, delegation for free, and events that follow the React tree (portals included). The native event stays available at e.nativeEvent.

Beginner: Difference between onClick={fn}, onClick={fn(id)}, and onClick={() => fn(id)}?

fn passes the handler (called with the event). fn(id) calls immediately at render — a bug unless fn returns a handler. () => fn(id) creates a handler that calls with your argument on click — the standard way to bind data.

Mid: Why doesn't React's onChange behave like the DOM change event?

React normalized it to the native input event: it fires on every value change (each keystroke), because controlled components (next lessons) need to re-render per keystroke to keep value in sync. For commit-on-blur semantics, use onBlur or debounce — native lazy change isn't exposed under that name.

Mid: How do you correctly implement close-on-outside-click for a popover?

React handlers only cover your own elements, so attach a native pointerdown listener on document in an effect (while open), check rootRef.current.contains(e.target), close if outside, and remove the listener in cleanup. Also handle Escape for accessibility. Don't rely on stopPropagation inside the popover — synthetic stopping doesn't halt the native document listener.

Senior: A click inside a portal-rendered modal triggers the parent card's onClick. Why, and what do you do?

Synthetic events propagate along the React tree, not the DOM tree — the modal is a React child of the card even though it renders into document.body. This is by design (context and events stay consistent). If the card shouldn't react, either stop propagation at the modal boundary (onClick={e => e.stopPropagation()} on the modal root) or — cleaner — don't make giant surfaces clickable; give the card an explicit button area instead.

Senior: When would you bypass React's event system entirely, and what discipline applies?

Global targets (window resize/keydown shortcuts, document outside-click), non-passive wheel/touch handlers where you must preventDefault scrolling, high-frequency streams you want to throttle before touching state (pointermove at 120Hz), and APIs outside the event system (observers, media events). Discipline: attach in useEffect, remove in cleanup, keep handlers stable or re-subscribe deliberately, and update state at animation-frame or debounced cadence so a native firehose doesn't become a render firehose.

function SearchOnEnter({ onSearch }) {
  const [q, setQ] = useState("");
  function onKeyDown(e) {
    if (e.key === "Enter") onSearch(q);
    if (e.key === "Escape") setQ("");
  }
  return <input value={q} onChange={e => setQ(e.target.value)} onKeyDown={onKeyDown} />;

6 (hard): Implement a draggable slider thumb with pointer events: drag updates a 0–100 value, keeps working when the pointer leaves the element mid-drag, and stops on release.
Solution 
function Slider({ value, onChange }) {
  const trackRef = useRef(null);

  function valueFrom(e) {
    const r = trackRef.current.getBoundingClientRect();
    return Math.round(Math.min(Math.max((e.clientX - r.left) / r.width, 0), 1) * 100);
  }
  function onPointerDown(e) {
    e.target.setPointerCapture(e.pointerId);   // THE trick: events keep coming to this
    onChange(valueFrom(e));                    // element even outside its bounds
  }
  function onPointerMove(e) {
    if (e.buttons) onChange(valueFrom(e));     // only while pressed
  }
  return (
    <div ref={trackRef} className="track"
         onPointerDown={onPointerDown} onPointerMove={onPointerMove}>
      <div className="thumb" style={{ left: `${value}%` }} />
    </div>
  );
}
// setPointerCapture replaces the old document-mousemove/mouseup dance entirely,
// and pointer events make it work for touch and pen with zero extra code.

// Fix 1 — narrow the state: keep hoveredId in the ROW (each row its own

🎯 Key Takeaways

  • React delegates all events to the root and dispatches SyntheticEvents along the component tree — consistent cross-browser, portal-aware, with e.nativeEvent as the raw escape hatch.
  • Handlers are references: bind arguments with arrows; the event's essentials (target.value) get captured before any await.
  • preventDefault cancels browser actions (submit/navigate); stopPropagation shields ancestors — different jobs, and synthetic stopping never halts native document listeners.
  • React's onChange = native input (per keystroke); focus/blur bubble; pointer events unify mouse/touch/pen and setPointerCapture owns drags.
  • Native addEventListener belongs in effects with guaranteed cleanup — for document/window targets, non-passive handlers, and high-frequency streams throttled before they touch state.
  • Real <button>/<form> elements carry the accessibility contract; every pointer interaction ships with its keyboard twin.
