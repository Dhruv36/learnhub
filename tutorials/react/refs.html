Refs & the DOM Effects & Lifecycle

What you'll master: useRef as a render-invisible box (the state/ref decision table), DOM refs for focus, measurement, and imperative APIs, forwarding refs through your own components, useImperativeHandle for curated component APIs, and callback refs. Prerequisites: State, The Render Cycle (commit timing — refs attach during commit).

Part 1 — The Box That Doesn't Render

const ref = useRef(initialValue);
// ref is { current: initialValue } — the SAME object every render (stable identity).
// Reading ref.current: never triggers anything.
// Writing ref.current: never triggers anything. No re-render, ever.

// State vs ref is one question: "should changing this REDRAW the UI?"

// The canonical non-DOM uses:
const intervalId = useRef(null);              // handle to clean up later
const isInFlight = useRef(false);             // gate logic without rendering (Data Fetching ex. 6)
const widget     = useRef(null);              // third-party class instance
const latest     = useRef(value);             // "latest value" bridge for stable callbacks
useEffect(() => { latest.current = value; }); // (the pattern from useEffect Mastery ex. 7)
Why not just a variable? A local resets every render (function re-runs); a module-level variable is shared by all instances of the component. useRef is the only per-instance, render-surviving, non-rendering slot. If you've written let x outside a component to "remember" something — that's the bug this hook exists to fix.

Part 2 — DOM Refs: the Escape Hatch to Real Nodes

function SearchPage() {
  const inputRef = useRef(null);

  useEffect(() => {
    inputRef.current.focus();                  // the DOM node, after commit
  }, []);

  return <input ref={inputRef} placeholder="Search…" />;
}
// Timeline: render (ref.current === null!) → commit: React assigns the node →
// effects run (ref.current is the ). That's why DOM refs are read in
// effects and handlers — NEVER during render, where they're null on first pass.

What legitimately needs a DOM ref — the things React's declarative model doesn't cover: focus/blur/selection, scrolling (scrollIntoView), measurement (getBoundingClientRect), media playback (video.play()), canvas contexts, integrating non-React libraries, and observers (Intersection/Resize). What does not: changing text, classes, styles, or structure — that's state's job, and imperative writes there fight reconciliation and lose on the next render.

// Measurement example — with the commit-timing lesson applied:
function AutoGrowTextarea(props) {
  const ref = useRef(null);
  useLayoutEffect(() => {                      // before paint: no height flicker
    ref.current.style.height = "auto";
    ref.current.style.height = ref.current.scrollHeight + "px";
  });                                          // every commit: content may have changed
  return <textarea ref={ref} {...props} />;
}

Part 3 — Refs Through Components: forwardRef

// ref is NOT a prop —  does nothing by default (function
// components have no instance to hand you). You must forward it explicitly:

const TextField = forwardRef(function TextField({ label, ...props }, ref) {
  return (
    <label>
      {label}
      <input ref={ref} {...props} />          {/* forward to the real node */}
    </label>
  );
});

// Now parents can do:
const emailRef = useRef(null);

Part 4 — useImperativeHandle: a Curated Handle, Not the Raw Node

// Exposing the raw DOM node leaks your internals — parents start calling
// anything on it. Expose a designed API instead:
const VideoPlayer = forwardRef(function VideoPlayer({ src }, ref) {
  const videoRef = useRef(null);

  useImperativeHandle(ref, () => ({
    play:  () => videoRef.current.play(),
    pause: () => videoRef.current.pause(),
    seek:  t  => { videoRef.current.currentTime = t; },
    // deliberately NOT exposed: the node itself, volume, playbackRate…
  }), []);

  return <video ref={videoRef} src={src} />;
});

// Parent sees only the contract:
playerRef.current.seek(42);      // ✅
playerRef.current.style;         // undefined — internals stay internal
// Use sparingly: if a parent needs play/pause CONSTANTLY, ask whether the state
// (playing) should be lifted and passed as a prop instead. Imperative handles
// are for commands that are awkward as state ("seek to 42s NOW"), not for data.

Part 5 — Callback Refs & Observing Nodes

// A ref can be a FUNCTION — React calls it with the node on attach, null on detach:
<li ref={node => {
  if (node) rowMap.current.set(item.id, node);   // attach
  else      rowMap.current.delete(item.id);      // detach (or return a cleanup — React 19)
}}>

// Killer use 1 — dynamic collections (a Map of row nodes for scroll-to-row):
function scrollTo(id) { rowMap.current.get(id)?.scrollIntoView({ block: "center" }); }

// Killer use 2 — "run when the node actually exists", even with conditional render:
// a plain ref + effect misses nodes mounted later; a callback ref fires exactly
// on mount/unmount of the node itself.

// Observers pair naturally with refs:
function useOnScreen(ref) {
  const [visible, setVisible] = useState(false);
  useEffect(() => {
    const obs = new IntersectionObserver(([e]) => setVisible(e.isIntersecting));
    if (ref.current) obs.observe(ref.current);
    return () => obs.disconnect();               // cleanup, as always
  }, [ref]);
  return visible;
}

Part 6 — The Rules That Keep Refs Safe (Senior Layer)

// 1. Don't read/write refs during render (except lazy-init):
if (!ref.current) ref.current = createExpensiveThing();   // ✅ the ONE sanctioned render write
const x = ref.current.value;                               // ❌ anywhere else in render

// 2. Refs are invisible to React: changing ref.current re-renders NOTHING.
//    If the UI must react to it, it was state all along — the most common ref misuse:
const count = useRef(0);
<button onClick={() => { count.current++; }}>{count.current}</button>   // ❌ never updates

// 3. DOM writes through refs must not fight React-managed attributes:
ref.current.className = "x";     // ❌ next render silently reverts it
ref.current.focus();             // ✅ focus isn't rendered — no conflict
// Rule: imperative calls (focus/scroll/play/measure) yes; property/attribute
// writes that JSX also controls, no.

// 4. Conditional rendering nulls refs: {open && 
} — r.current is
//    null when closed. Guard every access: r.current?.focus().

Interview Questions

Beginner: useRef vs useState — the decision rule?

Does changing the value need to update what the user sees? Yes → state (setter triggers re-render). No — it's bookkeeping like a timer id, an instance, a flag — → ref (mutable, persistent, render-silent). A ref is a per-instance box that survives renders without participating in them.

Beginner: When is ref.current actually set for a DOM ref?

During the commit phase, after React creates/updates the DOM. It's null during the first render's function execution — which is why DOM access belongs in effects (post-commit) or event handlers, never in the render body.

Mid: Why does <MyInput ref={r} /> not work for a plain function component, and what are the fixes?

Function components have no instance for the ref to point at, and ref historically wasn't passed through props. Fixes: wrap with forwardRef so the component receives the ref and attaches it to an inner node; optionally curate with useImperativeHandle; in React 19, ref arrives as a regular prop, retiring forwardRef.

Mid: Give the "previous value" pattern and explain the ordering that makes it work.

const prev = useRef(); useEffect(() => { prev.current = value; }); — during render, prev.current still holds last render's value (effects haven't run yet); after commit the effect overwrites it with the current one. Rendering reads old, effect writes new: a one-slot delay line. Used for "did X increase?", animation direction, transition detection.

Senior: Why are refs deliberately excluded from React's reactivity, and what discipline follows?

Refs are the sanctioned escape from the declarative model — a mutable cell whose writes React cannot see and must not care about, or every mutation would need tracking and the purity/concurrency model breaks. Discipline: renders never depend on ref contents (output must be a function of props/state); effects don't list ref.current in deps (not reactive — use callback refs for attach/detach); and anything the UI reflects graduates from ref to state. Refs carry how (mechanics), state carries what (appearance).

Senior: Integrate a jQuery-era imperative widget (init/destroy/setData) safely into React.

Container <div ref={hostRef}/> that React renders and then never touches inside. Mount effect: widget = new Widget(hostRef.current, options), store on an instance ref; cleanup: widget.destroy() and null the ref (Strict Mode's double cycle verifies the pair). Narrow effects translate prop changes to widget commands (widget.setData(data) on [data]). Widget events → React callbacks, subscribed in the mount effect, unsubscribed in cleanup. The boundary rule: inside the host div is the widget's DOM, outside is React's — neither crosses.

1 (easy): A "Copy link" button that selects the text of a read-only input and copies it, with a focus flash.
Solution 
function CopyLink({ url }) {
  const inputRef = useRef(null);
  function copy() {
    inputRef.current.select();                       // imperative: selection isn't state
    navigator.clipboard.writeText(url);
  }
  return (
    <>
      <input ref={inputRef} value={url} readOnly />
      <button onClick={copy}>Copy</button>
    </>
  );
}

3 (medium): Build a Stopwatch with Start/Stop/Reset. Decide ref vs state for: elapsed ms, interval id, running flag. Justify each.
Solution 
function Stopwatch() {
  const [elapsed, setElapsed] = useState(0);   // displayed → state
  const [running, setRunning] = useState(false); // buttons change appearance → state
  const intervalRef = useRef(null);            // bookkeeping only → ref

  function start() {
    if (intervalRef.current) return;
    const t0 = Date.now() - elapsed;
    intervalRef.current = setInterval(() => setElapsed(Date.now() - t0), 50);
    setRunning(true);
  }
  function stop() {
    clearInterval(intervalRef.current);
    intervalRef.current = null;
    setRunning(false);
  }
  function reset() { stop(); setElapsed(0); }

  useEffect(() => () => clearInterval(intervalRef.current), []);   // unmount safety

  return (
    <div>
      <b>{(elapsed / 1000).toFixed(1)}s</b>
      <button onClick={running ? stop : start}>{running ? "Stop" : "Start"}</button>
      <button onClick={reset}>Reset</button>
    </div>
  );
}
// elapsed/running drive pixels → state. The interval id drives nothing visible → ref.
4 (medium): Create usePrevious(value) and use it to show "▲ up" / "▼ down" next to a live price.
Solution 
function usePrevious(value) {
  const ref = useRef();
  useEffect(() => { ref.current = value; });   // writes AFTER render commits
  return ref.current;                           // so render reads last commit's value
}

function Price({ value }) {
  const prev = usePrevious(value);
  const dir = prev === undefined || prev === value ? "" : value > prev ? "▲ up" : "▼ down";
  return <span>${value} {dir}</span>;
}
5 (medium): Build a 4-box OTP input: typing a digit auto-focuses the next box, Backspace on an empty box focuses the previous.
Solution 
function OtpInput({ onComplete }) {
  const [digits, setDigits] = useState(["", "", "", ""]);
  const refs = useRef([]);                              // array of nodes via callback refs

  function setDigit(i, v) {
    if (!/^\d?$/.test(v)) return;
    const next = digits.map((d, j) => (j === i ? v : d));
    setDigits(next);
    if (v && i < 3) refs.current[i + 1].focus();        // imperative focus — the ref job
    if (next.every(d => d)) onComplete(next.join(""));
  }
  function onKeyDown(i, e) {
    if (e.key === "Backspace" && !digits[i] && i > 0) refs.current[i - 1].focus();
  }

  return digits.map((d, i) => (
    <input key={i} value={d} maxLength={1} inputMode="numeric"
           ref={node => { refs.current[i] = node; }}    // callback ref into an array
           onChange={e => setDigit(i, e.target.value)}
           onKeyDown={e => onKeyDown(i, e)} />
  ));
}
// Values are state (displayed); focus choreography is refs (imperative, invisible to render).
6 (hard): Design a ScrollSpy nav: a list of section links where the link of the section currently on screen is highlighted. Use a Map of callback refs + one IntersectionObserver.
Solution 
function ScrollSpy({ sections }) {          // sections: [{id, title, content}]
  const [activeId, setActiveId] = useState(sections[0]?.id);
  const nodes = useRef(new Map());
  const observer = useRef(null);

  useEffect(() => {
    observer.current = new IntersectionObserver(entries => {
      const hit = entries.find(e => e.isIntersecting);
      if (hit) setActiveId(hit.target.dataset.id);
    }, { rootMargin: "-40% 0px -55% 0px" });   // a band around the viewport center
    nodes.current.forEach(n => observer.current.observe(n));
    return () => observer.current.disconnect();
  }, [sections]);

  const attach = id => node => {              // callback-ref factory
    if (node) { nodes.current.set(id, node); observer.current?.observe(node); }
    else {
      const old = nodes.current.get(id);
      if (old) observer.current?.unobserve(old);
      nodes.current.delete(id);
    }
  };

  return (
    <div className="spy">
      <nav>{sections.map(s => (
        <a key={s.id} href={`#${s.id}`} className={s.id === activeId ? "active" : ""}>
          {s.title}</a>))}</nav>
      <main>{sections.map(s => (
        <section key={s.id} id={s.id} data-id={s.id} ref={attach(s.id)}>
          <h2>{s.title}</h2>{s.content}</section>))}</main>
    </div>
  );
}
// activeId is state (highlights a link); the node Map and observer are refs
// (machinery). Callback refs handle sections mounting/unmounting dynamically.
7 (hard): A ConfirmDialog must focus its Cancel button when opened and return focus to the trigger button when closed. Implement the focus contract with refs, and explain why this can't be pure state.
Solution 
function useConfirm() {
  const [open, setOpen] = useState(false);
  const triggerRef = useRef(null);            // who opened us
  const cancelRef  = useRef(null);            // initial focus target

  useEffect(() => {
    if (open) cancelRef.current?.focus();     // on open: move focus in
    else      triggerRef.current?.focus();    // on close: give it back
  }, [open]);

  return { open, setOpen, triggerRef, cancelRef };
}

function Page() {
  const { open, setOpen, triggerRef, cancelRef } = useConfirm();
  return (
    <>
      <button ref={triggerRef} onClick={() => setOpen(true)}>Delete…</button>
      {open && (
        <div role="dialog" aria-modal="true">
          <p>Really delete?</p>
          <button ref={cancelRef} onClick={() => setOpen(false)}>Cancel</button>
          <button onClick={confirmDelete}>Delete</button>
        </div>
      )}
    </>
  );
}
// Why not state: focus isn't an attribute you render — it's a one-shot ACTION on a
// node ("focus now"), and the DOM owns which element has it. React has no
// focused={true} prop; the imperative call via ref IS the API. The effect ties the
// actions to the open/close transitions. (This focus contract is a WCAG requirement
// for modals — see the HTML track's Accessibility lessons.)

🎯 Key Takeaways

  • useRef is a stable, mutable, render-invisible box: timer ids, instances, flags, previous values — anything that must persist without redrawing.
  • The decision rule: user-visible → state; bookkeeping → ref. Displaying ref.current in JSX is the canonical misuse.
  • DOM refs attach at commit: read them in effects/handlers (guarded — conditionals null them), never in render; imperative calls yes, React-managed attributes no.
  • Refs don't cross component boundaries by default: forwardRef (ref-as-prop in React 19), and useImperativeHandle to expose a curated API instead of raw DOM.
  • Callback refs fire on attach/detach — the tool for node collections (Maps) and observer wiring in dynamic lists.
  • Refs are non-reactive by design: never in dep arrays, never driving render output — they carry mechanics, state carries appearance.
