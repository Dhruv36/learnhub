Binary Search & Variants Core Pattern

Concept: Any time you can answer "is the answer ≤ x?" with a monotonic yes/no, you can binary search — not just on sorted arrays, but on answer spaces ("minimum capacity to ship packages in D days", "smallest divisor…"). Halving each step → O(log n).

Recognition Cues

🔍 Sorted (or rotated-sorted) input • "find first/last occurrence" • "minimize the maximum / maximize the minimum" • O(log n) required by constraints (n up to 10⁹) • monotonic condition ("if capacity C works, C+1 works too").

Template (the bug-free version)

let lo = 0, hi = n - 1;               // invariant: answer ∈ [lo, hi]
while (lo < hi) {
  const mid = lo + ((hi - lo) >> 1);  // no overflow, rounds down
  if (condition(mid)) hi = mid;       // mid could be the answer → keep it
  else lo = mid + 1;                  // mid can't be → discard it
}
return lo;                            // first index where condition is true

Why this template: it finds the boundary of a true/false region and never infinite-loops (the range strictly shrinks). 90% of binary search bugs come from mixing lo <= hi, hi = mid - 1 styles — pick one template and drill it.

Practice Problems

🟢 Easy — Binary Search (LC 704)

Classic: find target's index in a sorted array, or −1.

Show solution & analysis 
function search(nums, target) {
  let lo = 0, hi = nums.length - 1;
  while (lo <= hi) {
    const mid = lo + ((hi - lo) >> 1);
    if (nums[mid] === target) return mid;
    if (nums[mid] < target) lo = mid + 1;
    else hi = mid - 1;
  }
  return -1;
}

Complexity: O(log n), O(1). Exact-match search is the one case where the lo <= hi style is simplest.

🟡 Medium — Search in Rotated Sorted Array (LC 33)

[4,5,6,7,0,1,2], target 0 → index 4. Still must be O(log n).

Show solution & analysis 
function search(nums, target) {
  let lo = 0, hi = nums.length - 1;
  while (lo <= hi) {
    const mid = lo + ((hi - lo) >> 1);
    if (nums[mid] === target) return mid;
    if (nums[lo] <= nums[mid]) {                    // left half is sorted
      if (nums[lo] <= target && target < nums[mid]) hi = mid - 1;
      else lo = mid + 1;
    } else {                                        // right half is sorted
      if (nums[mid] < target && target <= nums[hi]) lo = mid + 1;
      else hi = mid - 1;
    }
  }
  return -1;
}

Key insight: after any rotation, at least one half is still sorted. Check which half, check if target lies inside it, and discard the other. O(log n), O(1).

🔴 Hard — Median of Two Sorted Arrays (LC 4)

Find the median of two sorted arrays in O(log(min(m,n))). The canonical hard binary search.

Show solution & analysis 
function findMedianSortedArrays(A, B) {
  if (A.length > B.length) [A, B] = [B, A];   // search on the smaller array
  const m = A.length, n = B.length;
  const half = (m + n + 1) >> 1;
  let lo = 0, hi = m;
  while (lo <= hi) {
    const i = (lo + hi) >> 1;                 // take i elements from A…
    const j = half - i;                       // …and j from B for the left part
    const Aleft  = i === 0 ? -Infinity : A[i - 1];
    const Aright = i === m ?  Infinity : A[i];
    const Bleft  = j === 0 ? -Infinity : B[j - 1];
    const Bright = j === n ?  Infinity : B[j];
    if (Aleft <= Bright && Bleft <= Aright) { // correct partition found
      if ((m + n) % 2) return Math.max(Aleft, Bleft);
      return (Math.max(Aleft, Bleft) + Math.min(Aright, Bright)) / 2;
    }
    if (Aleft > Bright) hi = i - 1;           // took too many from A
    else lo = i + 1;                          // took too few from A
  }
}

Insight: we binary search the partition point, not a value: split both arrays so the combined left part has exactly half the elements and every left element ≤ every right element. The ±Infinity sentinels remove all edge-case branching.
