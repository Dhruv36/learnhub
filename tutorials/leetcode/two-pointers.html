Two Pointers Core Pattern

Concept: Two indices walk through the data (toward each other, in the same direction, or across two arrays), shrinking the search space with every step. This replaces a nested loop, turning O(n²) brute force into O(n) with O(1) extra space.

Recognition Cues

🔍 Sorted array + "find a pair/triplet with sum X" • "in-place" removal or partitioning • palindrome checks (compare both ends) • merging or comparing two sorted sequences • "container/area between two lines".

Template (converging pointers)

let left = 0, right = arr.length - 1;
while (left < right) {
  const current = evaluate(arr[left], arr[right]);
  if (current === target) return answer;
  if (current < target) left++;   // need more → move the small side
  else right--;                    // need less → move the large side
}

Why it's correct: in a sorted array, moving left right can only increase the sum and moving right left can only decrease it — so each step safely discards every pair involving the abandoned element. No pair is ever missed.

Practice Problems

🟢 Easy — Valid Palindrome (LC 125)

Given a string, return true if it reads the same forward and backward, considering only alphanumeric characters and ignoring case. "A man, a plan, a canal: Panama" → true.

Show solution & analysis 
function isPalindrome(s) {
  let l = 0, r = s.length - 1;
  const ok = c => /[a-z0-9]/i.test(c);
  while (l < r) {
    while (l < r && !ok(s[l])) l++;          // skip punctuation/spaces
    while (l < r && !ok(s[r])) r--;
    if (s[l].toLowerCase() !== s[r].toLowerCase()) return false;
    l++; r--;
  }
  return true;
}

Complexity: O(n) time, O(1) space. Interview trap: don't preprocess with regex + reverse — that's O(n) extra space; the pointer version is what they want.

🟡 Medium — 3Sum (LC 15)

Return all unique triplets in the array that sum to zero. [-1,0,1,2,-1,-4] → [[-1,-1,2],[-1,0,1]].

Show solution & analysis 
function threeSum(nums) {
  nums.sort((a, b) => a - b);
  const res = [];
  for (let i = 0; i < nums.length - 2; i++) {
    if (i > 0 && nums[i] === nums[i - 1]) continue;   // skip duplicate anchors
    if (nums[i] > 0) break;                            // sorted: no zero-sum possible
    let l = i + 1, r = nums.length - 1;
    while (l < r) {
      const sum = nums[i] + nums[l] + nums[r];
      if (sum === 0) {
        res.push([nums[i], nums[l], nums[r]]);
        while (l < r && nums[l] === nums[l + 1]) l++;  // skip duplicate pairs
        while (l < r && nums[r] === nums[r - 1]) r--;
        l++; r--;
      } else if (sum < 0) l++;
      else r--;
    }
  }
  return res;
}

Complexity: O(n²) time (sort O(n log n) + n × two-pointer scan), O(1) extra space. Key insight: fix one element, reduce to Two Sum on the rest. The three duplicate-skips are where most candidates fail — practice them until automatic.

🔴 Hard — Trapping Rain Water (LC 42)

Given bar heights, compute how much rain water gets trapped between them. [0,1,0,2,1,0,1,3,2,1,2,1] → 6.

Show solution & analysis 
function trap(height) {
  let l = 0, r = height.length - 1;
  let leftMax = 0, rightMax = 0, water = 0;
  while (l < r) {
    if (height[l] < height[r]) {           // water at l is bounded by leftMax
      leftMax = Math.max(leftMax, height[l]);
      water += leftMax - height[l];
      l++;
    } else {                                // water at r is bounded by rightMax
      rightMax = Math.max(rightMax, height[r]);
      water += rightMax - height[r];
      r--;
    }
  }
  return water;
}

Complexity: O(n) time, O(1) space — beats the O(n)-space prefix-max arrays approach. Intuition: water above any bar = min(maxLeft, maxRight) − height. Since we always advance the shorter side, we already know the min bound — the taller side can't lower it.

When NOT to Use It

Unsorted data where sorting would destroy needed information (use a hash map — classic Two Sum), or when you need all subarrays/substrings matching a condition (that's Sliding Window).
