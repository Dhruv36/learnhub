Sliding Window Core Pattern

Concept: Maintain a window [left, right] over a contiguous range. Grow it by moving right; shrink it by moving left when the window becomes invalid. Each element enters and leaves at most once → O(n) instead of checking every subarray (O(n²/n³)).

Recognition Cues

🔍 "longest/shortest substring or subarray that…" • "maximum sum of subarray of size k" (fixed window) • "at most K distinct…" • "minimum window containing…" — anything about a contiguous range with a constraint.

Template (variable-size window)

let left = 0, best = 0;
const state = new Map();               // whatever tracks window validity
for (let right = 0; right < s.length; right++) {
  add(s[right], state);                // 1. expand
  while (!valid(state)) {              // 2. shrink until valid again
    remove(s[left], state);
    left++;
  }
  best = Math.max(best, right - left + 1);  // 3. record answer
}

Practice Problems

🟢 Easy — Maximum Average Subarray I (LC 643)

Find the contiguous subarray of exactly length k with the maximum average.

Show solution & analysis 
function findMaxAverage(nums, k) {
  let sum = 0;
  for (let i = 0; i < k; i++) sum += nums[i];   // first window
  let best = sum;
  for (let i = k; i < nums.length; i++) {
    sum += nums[i] - nums[i - k];               // slide: add new, drop old
    best = Math.max(best, sum);
  }
  return best / k;
}

Complexity: O(n) time, O(1) space. Fixed-size windows never recompute the whole sum — that's the entire trick: +new −old.

🟡 Medium — Longest Substring Without Repeating Characters (LC 3)

"abcabcbb" → 3 ("abc"). The most-asked interview question in existence.

Show solution & analysis 
function lengthOfLongestSubstring(s) {
  const last = new Map();               // char → last index seen
  let left = 0, best = 0;
  for (let right = 0; right < s.length; right++) {
    const c = s[right];
    if (last.has(c) && last.get(c) >= left) {
      left = last.get(c) + 1;           // jump past the duplicate
    }
    last.set(c, right);
    best = Math.max(best, right - left + 1);
  }
  return best;
}

Complexity: O(n) time, O(min(n, alphabet)) space. Optimization to mention: the map lets left jump directly instead of shrinking one-by-one — interviewers love hearing you point that out.

🔴 Hard — Minimum Window Substring (LC 76)

Find the smallest window in s containing every character of t (with counts). s="ADOBECODEBANC", t="ABC" → "BANC".

Show solution & analysis 
function minWindow(s, t) {
  const need = new Map();
  for (const c of t) need.set(c, (need.get(c) || 0) + 1);
  let missing = t.length;               // chars still needed
  let left = 0, start = 0, minLen = Infinity;
  for (let right = 0; right < s.length; right++) {
    const c = s[right];
    if (need.has(c)) {
      if (need.get(c) > 0) missing--;   // this char was still needed
      need.set(c, need.get(c) - 1);
    }
    while (missing === 0) {             // window valid → shrink for minimum
      if (right - left + 1 < minLen) { minLen = right - left + 1; start = left; }
      const d = s[left++];
      if (need.has(d)) {
        need.set(d, need.get(d) + 1);
        if (need.get(d) > 0) missing++; // window just became invalid
      }
    }
  }
  return minLen === Infinity ? "" : s.substr(start, minLen);
}

Complexity: O(|s| + |t|) time, O(alphabet) space. The missing counter avoids re-scanning the need-map on every step — that's what separates the O(n) solution from O(n·26).

When NOT to Use It

Non-contiguous subsequences (that's DP), or when the window validity isn't monotonic — e.g. with negative numbers, growing a window can both help and hurt a sum, so use prefix sums instead.
