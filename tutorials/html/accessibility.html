Accessibility (ARIA) Advanced

~16% of the world lives with a disability, accessibility lawsuits are routine, and — the selfish argument — accessible HTML is better HTML: more robust, more testable, better for SEO. The skill interviews check: knowing when you don't need ARIA.

Rule #1 of ARIA: Don't Use ARIA

<!-- ❌ Rebuilding a button — 4 attributes + JS keyboard handling, still worse -->
<div role="button" tabindex="0" aria-pressed="false" onclick="…" onkeydown="…">Save</div>

<!-- ✅ The real thing — keyboard, focus, semantics all built in -->
<button>Save</button>

Native elements ship with keyboard handling, focus behavior, and semantics for free. ARIA only fills gaps where no native element exists (tabs, comboboxes, live regions).

The ARIA You Actually Need

AttributeUse case
aria-labelName for icon-only buttons: <button aria-label="Close">✕</button>
aria-expandedOn the trigger of dropdowns/accordions: "collapsed" vs "expanded"
aria-current="page"Which nav link is the current page
aria-live="polite"Announce dynamic updates: "3 results found", toast messages
aria-hidden="true"Hide decorative icons from screen readers
role="alert"Urgent announcements (form errors) — interrupts immediately

Keyboard Navigation — the 20% That Catches 80% of Bugs

Tab       — next interactive element        Shift+Tab — previous
Enter     — activate links & buttons         Space     — buttons, checkboxes
Arrows    — radio groups, selects, tabs      Escape    — close modals/menus

Test: unplug your mouse and complete your app's core flow. Common failures: focus lost after a modal closes, no visible focus ring (:focus-visible in CSS), custom widgets unreachable by Tab. tabindex="0" adds an element to tab order; tabindex="-1" allows programmatic focus; positive values are always a bug.

The Skip Link (on every production site)

<body>
  <a href="#main" class="skip-link">Skip to content</a>  <!-- visually hidden until focused -->
  <header>… 40 nav links …</header>
  <main id="main">…</main>
💡 Production tip: automated checkers (Lighthouse, axe) catch only ~30% of accessibility issues — they can't tell if your alt text is meaningful or your focus order is logical. Ship-blocking a11y review at big companies = automated scan + keyboard walkthrough + screen-reader spot check (NVDA is free).
