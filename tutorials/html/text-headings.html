Text & Headings Basics

Headings are not font sizes — they are the outline of your document. Screen readers, search engines, and reader modes all navigate by them.

The Heading Hierarchy

<h1>Pricing</h1>                    <!-- ONE per page: the page's topic -->
  <h2>Plans</h2>
    <h3>Free tier</h3>
    <h3>Pro tier</h3>
  <h2>FAQ</h2>

Rules that hold in production: exactly one <h1>; never skip levels (h2 → h4 breaks the outline); never pick a heading tag because of its default size — that's CSS's job.

Paragraphs & Inline Semantics

<p>The deploy failed because the <strong>API key expired</strong> —
   see the <em>runbook</em> for rotation steps. Press
   <kbd>Ctrl+C</kbd> to abort, or run <code>npm run deploy</code> again.</p>
ElementMeaning (not just styling)
<strong>Importance — screen readers may add emphasis. Use over <b>
<em>Stress emphasis ("do not deploy on Friday")
<code>, <pre>Inline code / preformatted block (preserves whitespace)
<mark>, <kbd>, <time>Highlight, keyboard input, machine-readable dates
<blockquote cite="…">Quoted content with source

Entities — Displaying Reserved Characters

&lt;   →  <        &gt;   →  >        &amp;  →  &
&quot; →  "        &copy; →  ©        &nbsp; →  non-breaking space

Any time you show code or user-generated text inside HTML, < and & must be escaped — this becomes a security requirement in the XSS lesson, not just a rendering detail.

💡 Production tip: misuse of <br> is the classic beginner tell. It's for line breaks inside content (poems, addresses) — never for spacing between blocks. Spacing is CSS margin.
