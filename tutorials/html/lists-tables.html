Lists & Tables Basics

The Three List Types

<ul><li>Unordered — bullets (nav menus are almost always this)</li></ul>

<ol start="3" reversed><li>Ordered — steps, rankings</li></ol>

<dl>                                  <!-- description list: term/definition pairs -->
  <dt>QPS</dt><dd>Queries per second</dd>
  <dt>SLA</dt><dd>Service level agreement</dd>
</dl>

Lists nest: a <ul> inside an <li> makes a submenu. In production nav bars, the pattern <nav><ul><li><a> is universal — assistive tech announces "list, 5 items" so users know the menu's size before reading it.

Tables — For Data, Never Layout

<table>
  <caption>Q3 revenue by region</caption>
  <thead>
    <tr><th scope="col">Region</th><th scope="col">Revenue</th><th scope="col">Growth</th></tr>
  </thead>
  <tbody>
    <tr><th scope="row">EMEA</th><td>$4.2M</td><td>+12%</td></tr>
    <tr><th scope="row">APAC</th><td>$3.1M</td><td>+28%</td></tr>
  </tbody>
  <tfoot>
    <tr><th scope="row">Total</th><td>$7.3M</td><td>+19%</td></tr>
  </tfoot>
</table>
ElementPurpose
<caption>Announced title — what screen-reader users hear first
<thead>/<tbody>/<tfoot>Structure; enables sticky headers & repeated headers when printing
scope="col|row"Associates each cell with its headers — cell "$3.1M" reads as "APAC, Revenue, $3.1M"
colspan / rowspanMerge cells across columns/rows
💡 Production tip: wide tables must scroll inside their own container — <div style="overflow-x:auto"><table>…</div> — or they break mobile layouts. And if you're tempted to use a table to position things: that era ended in 2005; that's CSS Grid's job.
