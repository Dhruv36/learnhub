Performance Advanced

HTML decides most of your page's loading behavior before a line of JavaScript runs: what blocks rendering, what loads eagerly, what waits. Google ranks on these metrics (Core Web Vitals), and conversion studies consistently show ~1% revenue loss per 100ms of delay.

Script Loading — the Difference Between Fast and Frozen

<script src="app.js"></script>          ❌ blocks parsing while it downloads+runs
<script src="app.js" defer></script>    ✅ parallel download, runs after parse, keeps order
<script src="ads.js" async></script>    ⚠ runs the moment it arrives, any order — analytics only
<script type="module" src="app.mjs"></script>   ✅ modules defer by default

Resource Hints — Telling the Browser the Future

<link rel="preload" href="/fonts/inter.woff2" as="font" type="font/woff2" crossorigin>
<link rel="preload" href="/img/hero.avif" as="image">      <!-- fetch the LCP image NOW -->
<link rel="preconnect" href="https://api.myapp.com">        <!-- open TCP+TLS early -->
<link rel="dns-prefetch" href="https://cdn.thirdparty.com">
<link rel="prefetch" href="/checkout.html">                 <!-- likely NEXT page, idle-time fetch -->

preload = "needed for THIS page, fetch immediately" (fonts and hero images — both are discovered late by default). prefetch = "probably needed NEXT". Preloading too much cancels the benefit — budget 2–3 truly critical resources.

Lazy Loading — and Its One Big Trap

<img src="/img/below-fold.jpg" loading="lazy" …>     ✅ everything below the fold
<img src="/img/hero.jpg" fetchpriority="high" …>     ✅ the LCP image: NEVER lazy
<iframe src="…/embed" loading="lazy"></iframe>       ✅ heavy embeds (maps, videos)

Lazy-loading the hero image is the most common self-inflicted LCP regression in real codebases — the browser delays the one image users are waiting for.

Fonts Without the Flash

@font-face { font-family: Inter; src: url(/fonts/inter.woff2); font-display: swap; }

font-display: swap shows text in a fallback font immediately instead of invisible text (FOIT) while the webfont downloads. Pair with the preload above.

💡 Production tip: measure before optimizing — run Lighthouse (F12 → Lighthouse tab) and fix the top item only. The usual ranking of wins: compress/resize images → defer scripts → preload LCP image + fonts → prune third-party tags. Third-party scripts routinely cost more than all your own code combined.
