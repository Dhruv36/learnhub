Web Components & Shadow DOM Ultra-Advanced

The browser's native component model: define your own HTML elements with encapsulated markup, style, and behavior — no framework required. YouTube, GitHub, and Adobe Photoshop-on-web ship on these. Their superpower in production: components that work identically in React, Angular, and plain HTML — the standard answer to design systems in multi-framework companies.

A Custom Element in 20 Lines

class UserBadge extends HTMLElement {
  connectedCallback() {                       // runs when added to the DOM
    const shadow = this.attachShadow({ mode: "open" });
    shadow.innerHTML = `
      <style>
        /* Scoped: this CSS cannot leak out, page CSS cannot leak in */
        .badge { display: inline-flex; gap: .5rem; padding: .3rem .7rem;
                 border-radius: 999px; background: #eaf5ee; font: 14px system-ui; }
      </style>
      <span class="badge"><slot></slot></span>`;
  }
}
customElements.define("user-badge", UserBadge);
<user-badge>Dhruv · Staff Engineer</user-badge>   <!-- use it like any element -->

The Three Pillars

TechnologyWhat it gives you
Custom ElementsYour own tags with lifecycle hooks (connectedCallback, attributeChangedCallback)
Shadow DOMStyle + DOM encapsulation — the component can't be broken by page CSS (the eternal "global CSS collision" problem, solved)
Slots / Templates<slot> = projection points for user content; <template> = inert reusable markup

Reacting to Attributes

static observedAttributes = ["status"];
attributeChangedCallback(name, oldVal, newVal) {
  this.shadowRoot.querySelector(".badge").dataset.status = newVal;
}
<user-badge status="online">Dhruv</user-badge>

Styling Hooks — Encapsulation With a Public API

/* Inside the component: expose knobs via custom properties */
.badge { background: var(--badge-bg, #eaf5ee); }

/* Outside: consumers style ONLY what you exposed */
user-badge { --badge-bg: #fff8c5; }
user-badge::part(label) { font-weight: 700; }   /* if you added part="label" */
💡 Staff-level insight: Web Components vs React isn't a war — they solve different layers. Frameworks own application state and data flow; web components excel at leaf-level design-system primitives shared across teams and stacks. Big-company pattern: design system in web components (or Lit), products in whatever framework each team prefers.
