Links & Images Basics

Links

<a href="/pricing">Pricing</a>                        <!-- same site: absolute path -->
<a href="../docs/api.html">API docs</a>               <!-- relative to current file -->
<a href="#faq">Jump to FAQ</a>                        <!-- anchor on this page -->
<a href="mailto:team@app.com">Email us</a>
<a href="https://partner.com" target="_blank" rel="noopener noreferrer">Partner ↗</a>
RuleWhy
Descriptive link text, never "click here"Screen-reader users tab through links out of context; SEO reads anchor text
rel="noopener" with target="_blank"Stops the new tab from controlling your page via window.opener (tabnabbing)
Root-relative (/docs) over file-relative for app linksSurvives page moves; file-relative breaks when the linking page relocates

Images

<img src="/img/chart.png"
     alt="Revenue grew 40% quarter over quarter in 2025"
     width="800" height="450"
     loading="lazy">

Every attribute above earns its place in production:

alt — describes the image's meaning. If purely decorative, use alt="" (empty, but present) so screen readers skip it instead of reading the filename.
width/height — lets the browser reserve space before the image loads, preventing layout shift (a Core Web Vitals metric).
loading="lazy" — below-the-fold images load only when scrolled near.

Responsive Images (what real products ship)

<picture>
  <source srcset="/img/hero.avif" type="image/avif">
  <source srcset="/img/hero.webp" type="image/webp">
  <img src="/img/hero.jpg" alt="…" width="1200" height="600">
</picture>

The browser picks the first format it supports — AVIF/WebP are 30–50% smaller than JPEG. srcset with widths (hero-400.jpg 400w, hero-800.jpg 800w) additionally serves phones smaller files than desktops.

Figures

<figure>
  <img src="/img/arch.png" alt="Service architecture diagram">
  <figcaption>Fig 1. Payment flow through the gateway.</figcaption>
</figure>
💡 Production tip: missing width/height on images is the #1 cause of Cumulative Layout Shift — the page "jumps" as images pop in. Set both, always; CSS max-width: 100%; height: auto keeps them responsive anyway.
