HTML Introduction Basics

HTML (HyperText Markup Language) is the structure of every web page in existence. Browsers don't run designs or ideas — they parse HTML into a tree (the DOM) and paint it. Everything else you'll learn (CSS, JavaScript, React) is a layer on top of this tree.

Your First Page

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello, World!</h1>
    <p>My first web page.</p>
  </body>
</html>

Three parts matter: <!DOCTYPE html> tells the browser to use modern standards mode; <head> holds metadata the user never sees; <body> holds everything they do.

An element = opening tag + content + closing tag. Attributes configure it. Some elements are self-closing ("void"): <img>, <br>, <input>, <meta>.

How Browsers Actually Read It

The browser parses your markup into the DOM tree. Bad nesting doesn't crash — the parser silently "fixes" it, often not the way you meant. That forgiveness is why validating your HTML matters: what renders fine today may break when JavaScript queries the mis-nested tree.

💡 Production tip: always declare lang on <html> and <meta charset="UTF-8"> first in head. Screen readers pick pronunciation rules from lang; the charset declaration prevents garbled text and must appear within the first 1024 bytes.
