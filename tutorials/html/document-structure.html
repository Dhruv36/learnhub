Document Structure Basics

A production HTML document is more than doctype + body. Here is the head every real product ships, line by line.

The Production-Ready Head

<head>
  <meta charset="UTF-8">                                       <!-- always first -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Dashboard · MyApp</title>
  <meta name="description" content="Track your team's metrics in real time.">
  <link rel="icon" href="/favicon.ico">
  <link rel="stylesheet" href="/css/app.css">
  <script src="/js/app.js" defer></script>
</head>
LineWhy it exists
viewportWithout it, phones render at 980px and zoom out — your "responsive" CSS never triggers
titleBrowser tab, bookmarks, and the blue link in Google results
descriptionThe gray snippet under that blue link
deferDownloads JS in parallel but runs it after parsing — never block rendering with bare <script src> in head

Body Skeleton (the layout every site shares)

<body>
  <header>  … logo, navigation …            </header>
  <main>    … the page's unique content …   </main>
  <footer>  … links, copyright …            </footer>
</body>

One <main> per page. Screen-reader users jump straight to it with a single key — that only works if it exists.

Comments & Whitespace

HTML collapses runs of whitespace into one space when rendering, so indent freely. Comments ship to every visitor — never leave secrets, internal URLs, or snark in them.

💡 Production tip: the <title> pattern "Page · Product" (not the reverse) keeps the useful part visible in narrow browser tabs — check any tab from GitHub or Stripe.
