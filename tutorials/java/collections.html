Collections Framework Basics

The Collections Framework is Java's data-structure library, organized around interfaces (List, Set, Map, Queue) with multiple implementations. Choosing the right one — and knowing its Big-O — is a constant interview and production concern. Program to the interface, pick the implementation for the access pattern.

Everyday Usage

List<String> names = new ArrayList<>();     // program to the INTERFACE (List), not ArrayList
names.add("Asha"); names.get(0); names.size();
Map<String, Integer> scores = new HashMap<>();
scores.put("Asha", 92);
scores.getOrDefault("x", 0);                     // safe default (no null)
scores.computeIfAbsent("group", k -> new ArrayList<>()).add(item);   // grouping idiom
scores.merge("Asha", 1, Integer::sum);           // increment-or-init

Set<String> unique = new HashSet<>(names);       // dedup + O(1) membership
for (var e : scores.entrySet()) { e.getKey(); e.getValue(); }   // iterate a map correctly
List<Integer> immutable = List.of(1, 2, 3);       // immutable factory (throws on modify)

The equals/hashCode Dependency (from the OOP lesson)

// HashMap/HashSet find elements by hashCode THEN equals. If your key class overrides
// equals but not hashCode (or has broken ones), lookups FAIL — the object "isn't there":
map.put(new Point(1,2), "a");
map.get(new Point(1,2));       // null if Point lacks proper equals/hashCode!
// Use records for value keys (auto equals/hashCode), or override both. Also: never MUTATE
// a key after inserting it into a hash structure — its bucket won't match anymore.

Iteration & Modification

// ConcurrentModificationException: modifying a collection while iterating it:
for (var s : list) if (bad(s)) list.remove(s);   // ❌ throws
list.removeIf(this::bad);                          // ✅ the correct removal idiom
var it = list.iterator();                          // ✅ or explicit iterator.remove()
while (it.hasNext()) { if (bad(it.next())) it.remove(); }
Collections.sort(list);   list.sort(comparator);   // sorting
Collections.unmodifiableList(list);                // read-only view

Interview Questions

Q1: ArrayList vs LinkedList — when is LinkedList actually better?

Almost never in practice. ArrayList (backed by an array) gives O(1) random access and cache- friendly iteration; LinkedList (nodes) gives O(1) insert/remove at a KNOWN position but O(n) get(i) and poor cache locality. LinkedList only wins for frequent add/remove at the ends via its Deque interface — and ArrayDeque beats it there too. Default to ArrayList; measure before choosing LinkedList.

Q2: How does HashMap work, and how do equals/hashCode participate?

Keys are placed in buckets by hashCode (then spread); within a bucket, equals distinguishes keys. get() hashes the key to a bucket then equals-compares candidates. So a key class needs a good hashCode (distributes) AND a consistent equals — override both or lookups fail. Collisions chain (and become a balanced tree past a threshold in modern JDKs). Mutating a key after insertion breaks the invariant (wrong bucket).

Q3: How do you safely remove elements while iterating?

Don't modify the collection through its own methods during a for-each — the iterator detects structural modification and throws ConcurrentModificationException (fail-fast). Use collection.removeIf(predicate) (cleanest), or an explicit Iterator and iterator.remove(). For concurrent access across threads, use the java.util.concurrent collections (ConcurrentHashMap, CopyOnWriteArrayList) instead.

🏋️ Practice: group a list of users by role with computeIfAbsent; count word frequencies with merge; dedup a list via HashSet; trigger ConcurrentModificationException then fix with removeIf; break a HashMap by using a key class without hashCode, then fix with a record; benchmark ArrayList.get vs LinkedList.get.

🎯 Key Takeaways

  • Program to interfaces (List/Map/Set); default impls: ArrayList, HashMap, HashSet, ArrayDeque.
  • Know the Big-O: ArrayList get O(1)/mid-insert O(n); HashMap O(1); TreeMap O(log n) sorted.
  • Hash structures depend on correct equals+hashCode; records give both; never mutate keys after insert.
  • Modify while iterating via removeIf/iterator.remove — not the collection's own methods (CME).
  • Prefer ArrayDeque over legacy Stack/Vector; List.of/Map.of for immutable collections.
