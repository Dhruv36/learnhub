Back-of-Envelope Estimation Fundamentals

The first 5 minutes of every design interview: turn vague requirements into numbers that justify your architecture. Nobody expects precision — they expect defensible orders of magnitude.

Numbers You Must Know

OperationLatencyTakeaway
L1 cache reference~0.5 nsMemory is nanoseconds
Main memory reference~100 ns
Read 1 MB from memory~250 µs
SSD random read~150 µsDisk is microseconds–milliseconds
Read 1 MB from SSD~1 ms
Round trip same datacenter~0.5 msNetwork dominates; cross-region is 100–300×
Round trip US ↔ Europe~150 ms

Cheat-Sheet Conversions

1 day ≈ 86,400 s ≈ 10⁵ s          (the single most useful approximation)
1 million requests/day ≈ 12 req/s
2³⁰ ≈ 10⁹ (1 GB)   2⁴⁰ ≈ 10¹² (1 TB)
Char = 1 B · Int = 4 B · UUID = 16 B · Avg tweet ≈ 140 B · Avg image ≈ 300 KB

Worked Example: Estimate Twitter

Assumptions: 300M MAU, 50% daily, each user posts 2 tweets/day, 10% contain media (1 MB)

Write QPS:  150M users × 2 tweets / 10⁵ s        ≈ 3,000 tweets/s   (peak ×2 ≈ 6,000)
Read QPS:   read:write ≈ 100:1                    ≈ 300,000 reads/s  → caching is non-negotiable
Text storage/day:   300M tweets × ~200 B          ≈ 60 GB/day
Media storage/day:  30M × 1 MB                    ≈ 30 TB/day → object storage + CDN, not a DB
5-year media total: 30 TB × 365 × 5               ≈ ~55 PB   → must discuss tiering/compression
💡 Staff-level insight: each number should trigger a design decision. 300K read QPS → cache + fan-out strategy. 30 TB/day of media → S3 + CDN, never in Postgres. If your estimate doesn't change your design, you estimated the wrong thing.

The 4-Step Method

1. Clarify scale (users, activity rate). 2. Compute QPS (writes, then reads via a stated ratio). 3. Compute storage (per-item size × volume × retention). 4. State the design consequences out loud.

Interview Follow-ups to Expect

• What's your peak-to-average ratio and why? (2–3× typical, 10× for event-driven spikes)
• How much of that read traffic can a cache absorb? (80–95% for social feeds)
• How many servers is 6,000 write QPS? (~10–20 modest app servers — show you know a single box handles 500–1K QPS)
