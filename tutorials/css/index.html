Selectors & Specificity Basics

CSS is a rule engine: selectors decide which elements, declarations decide what happens, and specificity decides who wins when rules conflict. Most "CSS is broken!" moments are specificity fights — this lesson ends those.

The Selector Toolbox

p            { }   /* element */
.card        { }   /* class — your workhorse */
#signup      { }   /* id — unique per page */
.card p      { }   /* descendant: any p inside .card */
.card > p    { }   /* child: direct children only */
.btn:hover   { }   /* state pseudo-classes: :hover :focus :disabled :checked */
li:first-child, li:nth-child(2n) { }   /* structural */
input[type="email"] { }               /* attribute */
.card::before { content: "★"; }        /* pseudo-elements create sub-parts */

Specificity — The Scoring System

Score as (ID, CLASS, ELEMENT):
p                    → (0,0,1)
.card p              → (0,1,1)
.card .title         → (0,2,0)   ← beats the one above
#main .card          → (1,1,0)   ← beats everything above
style="…"            → inline: beats all selectors
!important           → nuclear: beats inline (and starts wars)

Compare left to right — one ID outranks any number of classes. When scores tie, the later rule in the file wins (the "cascade" in Cascading Style Sheets).

The Production Strategy: Keep Specificity FLAT

/* ❌ Specificity arms race */
#sidebar div.widget ul li a.link { color: red; }    /* (1,3,3) — good luck overriding */

/* ✅ One class, one job */
.widget-link { color: red; }                         /* (0,1,0) — anyone can override */

Every large codebase converges on this rule: style with single classes, avoid IDs in CSS, reserve !important for utility classes that must always win (e.g. .hidden). Modern escape hatch: :where(.card) p has specificity (0,0,1) — :where() contributes zero, letting design systems ship overridable defaults.

💡 Production tip: debugging "why isn't my style applying?" — F12 → Elements → Styles panel shows every rule matching the element, crossed out when beaten, with the winner on top. It answers the question in 10 seconds; guessing never does.
