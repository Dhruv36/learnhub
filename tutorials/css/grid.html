CSS Grid Intermediate

Grid lays out in two dimensions at once — rows AND columns — and inverts flexbox's philosophy: the container defines a track structure, and items slot into it. Page shells, dashboards, galleries, forms with aligned labels: Grid. It's also the layout system with the best developer tooling in the browser — build with the DevTools grid inspector open.

Tracks, Lines, Cells, Areas — the Vocabulary

grid-template-columns: 240px 1fr 300px;
grid-template-rows: 64px auto 1fr;

     line 1      line 2         line 3        line 4      ← LINES (n tracks ⇒ n+1 lines)
       │  240px    │     1fr      │    300px    │
       ├───────────┼──────────────┼─────────────┤
       │   cell    │     cell     │    cell     │          ← TRACKS are the columns/rows;
       ├───────────┼──────────────┼─────────────┤            CELLS their intersections;
                                                              an AREA is any rectangle of cells.
Negative line numbers count from the end: -1 is the LAST line — grid-column: 1 / -1 = full width.

Sizing Tracks — fr, minmax, and Friends

The One-Liner That Replaced Media Queries

.cards {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(220px, 1fr));
  gap: 1rem;
}
/* As many ≥220px columns as fit; extras share leftover space; wraps automatically.
   Fully responsive card grid, ZERO breakpoints. (This site's homepage does exactly this.) */

auto-fill vs auto-fit — identical until there are fewer items than columns: auto-fit collapses the empty tracks so 2 cards stretch across a wide container; auto-fill keeps invisible "ghost" tracks so 2 cards stay card-sized. Sparse content that should stay compact → auto-fill; content that should always fill the row → auto-fit.

Placing Items — Three Levels of Control

/* 1. By line numbers (or spans): */
.hero   { grid-column: 1 / 3; grid-row: 1 / 3; }     /* from line to line */
.footer { grid-column: 1 / -1; }                       /* full width, track-count-proof */
.ad     { grid-column: span 2; }                       /* width in tracks, position auto */

/* 2. By NAMED AREAS — the layout is drawn in the CSS: */
.layout {
  display: grid;
  grid-template-columns: 240px 1fr 300px;
  grid-template-areas:
    "header header header"
    "nav    main   aside"
    "footer footer footer";
}
.header { grid-area: header; }  .nav { grid-area: nav; }   /* '.' in the map = empty cell */

/* 3. Responsive = REDRAW THE MAP, touch nothing else: */
@media (max-width: 800px) {
  .layout { grid-template-columns: 1fr;
            grid-template-areas: "header" "nav" "main" "aside" "footer"; }
}

The Implicit Grid — Where Extra Items Go

.gallery {
  grid-template-columns: repeat(4, 1fr);   /* explicit: 4 columns */
  grid-auto-rows: 180px;                   /* every IMPLICIT row (created by overflowing content) = 180px */
  grid-auto-flow: row;                     /* fill direction; 'dense' backfills holes (order scrambles!) */
}

Items beyond your defined rows don't vanish — the grid grows implicit tracks, sized auto unless grid-auto-rows/columns says otherwise. Unsized implicit rows are why "my grid rows are all different heights" happens.

Alignment — the Six Properties, One System

justify-* = inline (row) axis  ·  align-* = block (column) axis
*-items   = content WITHIN cells        place-items: center   ← 2D centering in 2 words
*-content = the whole TRACK GRID within the container (when tracks don't fill it)
*-self    = per-item override

subgrid — Cross-Card Alignment

.cards { display: grid; grid-template-columns: repeat(3, 1fr); gap: 1rem; }
.card  {
  display: grid;
  grid-template-rows: subgrid;   /* adopt the PARENT's row tracks */
  grid-row: span 3;              /* my title/body/button occupy 3 shared rows */
}
/* Result: every card's button row aligns across the whole grid, regardless of text length —
   the 'aligned card internals' problem that was previously impossible without JS or fixed heights. */

Interview Questions

Q1: Explain fr vs % vs auto in one breath.

% is a fraction of the container (blind to gaps and siblings); fr is a share of the space REMAINING after fixed tracks and gaps are paid; auto is content-sized. That's why fr + gap never overflows but % + gap does, and why mixed templates (240px 1fr) are natural in grid.

Q2: auto-fill vs auto-fit — when does the difference show?

Only when items are fewer than the columns that fit: auto-fit collapses empty tracks (items stretch); auto-fill keeps ghost tracks (items stay their minmax size). Card grids that shouldn't produce two comically wide cards on ultrawide monitors → auto-fill.

Q3: What does subgrid solve that nesting can't?

Nested grids define independent tracks — a child's rows can't line up with its cousins' across cards with different content. subgrid makes children adopt the parent's tracks, so internals (titles, prices, buttons) align across the whole layout. Before it: fixed heights or JS measurement.

Q4: How would you build a holy-grail layout that reorders on mobile — and what's the a11y caveat?

grid-template-areas with a redrawn map per breakpoint — the reorder is just a different area map. Caveat: reading/tab order still follows the DOM; if mobile order should differ MEANINGFULLY, reorder the DOM (or accept the mismatch only for non-interactive content).

🏋️ Practice (Playground):
1. The responsive card grid one-liner — resize and count columns; swap auto-fill ↔ auto-fit with only 2 cards and observe.
2. Holy-grail shell with named areas + a mobile map in one media query.
3. Break a 1fr track with a 400-character word; fix with minmax(0,1fr).
4. 12-column grid: place a hero (span 8) and sidebar (span 4), then a full-width footer via 1 / -1.
5. Three pricing cards with subgrid-aligned button rows (fallback: min-height) — compare with content of very different lengths.

🎯 Key Takeaways

  • Tracks/lines/cells/areas; negative lines count from the end; 1 / -1 = full span.
  • fr distributes leftover space (gap-aware); minmax(0,1fr) is the blowout-proof workhorse.
  • repeat(auto-fill, minmax(Npx, 1fr)) = responsive grids without media queries; fill vs fit differs on sparse content.
  • Named areas make the stylesheet its own diagram — responsive = redraw the map.
  • Implicit tracks catch overflow items — size them with grid-auto-rows; dense backfill scrambles order.
  • subgrid aligns card internals across the layout; place-items: center is 2-word centering.
  • Grid = container-driven 2D; Flex = content-driven 1D; compose them.
