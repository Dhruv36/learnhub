Cross-browser & Accessibility Ultra-Advanced

The last mile of production CSS: working for users who aren't on the latest Chrome with perfect vision and a mouse. This is where legal risk, real-device bugs, and inclusive design meet.

Graceful Degradation with @supports

/* Baseline that works everywhere: */
.gallery { display: flex; flex-wrap: wrap; gap: 1rem; }

/* Enhancement where supported: */
@supports (grid-template-rows: subgrid) {
  .gallery { display: grid; grid-template-columns: repeat(3, 1fr); }
}

Strategy: baseline works for everyone, enhancements layer on. Check support data on caniuse.com and set an explicit browser-support policy (e.g. "last 2 versions + Safari 15") — "works on my machine" is not a policy. Vendor prefixes are autoprefixer's job at build time, never handwritten.

Safari & Mobile Realities (the usual suspects)

height: 100dvh;        /* not 100vh — vh ignores iOS's collapsing URL bar */
padding-bottom: env(safe-area-inset-bottom);   /* iPhone home-bar notch */
-webkit-tap-highlight-color: transparent;      /* the gray tap flash */
input { font-size: 16px; }                     /* <16px triggers iOS auto-zoom on focus */

Accessibility CSS — the Checklist Auditors Use

RequirementImplementation
Contrast ≥ 4.5:1 (text), 3:1 (large/UI)Check every token pair in both themes (DevTools color picker shows the ratio)
Visible focus:focus-visible { outline: 2px solid var(--brand); outline-offset: 2px; } — never outline: none without replacement
Respect motion preference@media (prefers-reduced-motion: reduce) kill-switch (Transitions lesson)
Text zoom to 200%rem-based sizes; no fixed-height text containers that clip
Touch targets ≥ 44×44pxpadding on small icons/links, not just bigger glyphs
Don't communicate by color aloneerror = icon + message + color, not red alone

The .sr-only Utility (in every production codebase)

.sr-only {   /* visible to screen readers, invisible on screen */
  position: absolute; width: 1px; height: 1px;
  margin: -1px; padding: 0; border: 0;
  clip-path: inset(50%); overflow: hidden; white-space: nowrap;
}
<button><svg aria-hidden="true">…</svg><span class="sr-only">Search</span></button>

Companions: forced-colors: active (Windows High Contrast mode strips your colors — test it), and color-scheme: light dark on :root so native widgets (scrollbars, form controls) match your theme.

💡 Staff-level insight: accessibility scales the same way design systems do — bake it into the component library once (focus states, contrast-checked tokens, motion guards) and every product team inherits compliance. Auditing 40 product screens is remediation; auditing 12 components is engineering.
