Custom Properties & Theming Advanced

Custom properties (CSS variables) are real, live values in the cascade — they inherit, they can be scoped, and they can change at runtime. They are the mechanism behind every dark mode and every design token system you've used.

Basics

:root {                        /* global scope */
  --brand: #2f8d46;
  --space-3: 1rem;
  --radius: 8px;
}
.card {
  background: var(--surface, #fff);   /* second arg = fallback */
  border-radius: var(--radius);
  padding: var(--space-3);
}

Dark Mode — the Production Pattern

:root {
  --bg: #ffffff;  --ink: #1f2328;  --surface: #f6f8fa;
}
@media (prefers-color-scheme: dark) {          /* follow the OS… */
  :root { --bg: #0d1117; --ink: #e6edf3; --surface: #161b22; }
}
:root[data-theme="dark"]  { --bg: #0d1117; --ink: #e6edf3; --surface: #161b22; }
:root[data-theme="light"] { --bg: #ffffff; --ink: #1f2328; --surface: #f6f8fa; }
/* …but let the user's explicit toggle (JS sets data-theme) win over the OS */

body { background: var(--bg); color: var(--ink); }   /* components never mention colors */

The key insight: components reference semantic variables (--bg, --surface) — themes swap the values, and zero component CSS changes.

Scoped Variables — Component APIs

.badge  { background: var(--badge-bg, #eaf5ee); }    /* component exposes a knob */
.header .badge { --badge-bg: #fff3cd; }               /* context overrides it */

Runtime Superpowers (things Sass can't do)

/* JS can read/write them: */
document.documentElement.style.setProperty("--accent", userColor);

/* They compose with calc(): */
.stack > * + * { margin-top: calc(var(--space-3) * var(--density, 1)); }

/* Sass variables compile away at build time; custom properties live in the browser.
   Production codebases use both: Sass for structure, custom props for anything dynamic. */

Design Tokens — Naming That Scales

/* Tier 1 — raw palette (never used directly): */   --green-600: #2f8d46;
/* Tier 2 — semantic (what components use):     */   --color-action: var(--green-600);
/* Tier 3 — component-specific (optional):      */   --btn-bg: var(--color-action);

Rebranding = editing tier 1. Dark mode = swapping tier 2. Components never change. This is exactly how Material, Polaris (Shopify), and Primer (GitHub) are built.

💡 Staff-level insight: the discipline that makes theming work is banning raw values in components — enforce with a stylelint rule that rejects hex colors outside the tokens file. Design consistency stops being a review comment and becomes a build failure.
