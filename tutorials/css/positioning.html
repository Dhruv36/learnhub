Positioning Basics

Five values of position decide whether an element flows with the document or escapes it. Modern layout is flex/grid — positioning is for the exceptions: badges, dropdowns, sticky headers, modals.

The Five Values

ValueBehaviorClassic use
staticDefault — in flow, offsets ignoredEverything normal
relativeIn flow, but offsettable; becomes anchor for absolute childrenThe "anchor" wrapper
absoluteOut of flow; positions against nearest non-static ancestorBadges, dropdowns, tooltips
fixedOut of flow; pinned to the viewportChat bubbles, cookie banners
stickyIn flow until scrolled to its offset, then pinsTable headers, section navs (this site's sidebar)

The Pattern You'll Write 1000 Times

.avatar-wrapper { position: relative; }          /* 1. anchor the parent */
.notification-dot {
  position: absolute;                             /* 2. position the child */
  top: -4px; right: -4px;
  width: 12px; height: 12px;
  border-radius: 50%; background: red;
}

Forgetting step 1 is the classic bug: the dot then anchors to the nearest positioned ancestor — often <body> — and lands in a corner of the page.

Centering (the interview joke that's actually easy now)

/* Modern: */         .parent { display: grid; place-items: center; }
/* Flexbox: */        .parent { display: flex; justify-content: center; align-items: center; }
/* Absolute overlay:*/ .child { position: absolute; inset: 0; margin: auto; width: fit-content; height: fit-content; }

z-index & Stacking Contexts (where seniors get tested)

.modal { z-index: 9999; }   /* …and yet it's UNDER the header. why? */

z-index only competes within the same stacking context. A parent with transform, opacity < 1, filter, or its own z-index creates a new context — children can never escape it, no matter how large their z-index. Fixes: restructure the DOM (portals in React), or lift the z-index to the context that's actually competing.

💡 Production tip: define a z-index scale once (--z-dropdown: 100; --z-sticky: 200; --z-modal: 300; --z-toast: 400) and forbid raw numbers. Every mature codebase does this after its first "z-index: 999999" incident.
