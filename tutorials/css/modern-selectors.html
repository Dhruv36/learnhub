Layers, :has() & Modern Selectors Advanced

Three additions that rewired how production CSS is organized: cascade layers end specificity wars architecturally, :has() is the "parent selector" we waited 25 years for, and :is()/:where() make complex selectors maintainable.

Cascade Layers — Specificity by Policy, Not Accident

@layer reset, base, components, utilities;   /* declare priority order ONCE */

@layer reset      { * { margin: 0; } }
@layer base       { h1 { font-size: 2rem; } }
@layer components { .btn { padding: .6rem 1.2rem; } }
@layer utilities  { .hidden { display: none; } }

The rule: a later layer beats an earlier one regardless of selector specificity. A simple .hidden in utilities beats #app .modal .content in components. Third-party CSS goes in a low layer and can never fight your components again. Unlayered styles beat all layers — your escape hatch.

:has() — Style Parents by Their Children

.card:has(img)              { padding: 0; }            /* card WITH an image */
.field:has(input:invalid)   { border-color: red; }     /* wrap error styling, no JS */
.form:has([type="checkbox"]:checked) .submit { opacity: 1; }
label:has(+ input:focus)    { color: var(--brand); }   /* label BEFORE a focused input */
body:has(dialog[open])      { overflow: hidden; }      /* lock scroll when modal open */

Each line replaces a JS event listener + class toggle. Form UX, modal side-effects, and content-dependent layout became pure CSS.

:is() and :where() — Grouping Without Repetition

/* Before: */
.article h2, .article h3, .aside h2, .aside h3 { margin-top: 2rem; }
/* After: */
:is(.article, .aside) :is(h2, h3) { margin-top: 2rem; }

/* :where() = same grouping, ZERO specificity — perfect for library defaults: */
:where(button, [role="button"]) { cursor: pointer; }   /* trivially overridable */

Gotcha: :is() takes the specificity of its most specific argument; :where() is always (0,0,0). Design systems ship defaults in :where() so consumers override with a single class.

Honorable Mentions You'll Actually Use

.grid > :nth-child(odd of .visible) { }   /* nth of a FILTERED set */
input:user-invalid { }                     /* invalid only AFTER the user touched it */
.item:focus-within { }                     /* any descendant focused */
@supports selector(:has(a)) { }            /* feature-detect selectors themselves */
💡 Staff-level insight: :has() is powerful but not free — broad patterns like div:has(.foo) force the engine to re-check ancestors on every DOM mutation. Keep the anchor selector narrow (.card:has(...), not *:has(...)). Layers + :where() defaults + tokens (previous lesson) = the modern CSS architecture stack.
