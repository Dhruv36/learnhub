Transitions & Animations Intermediate

Motion communicates: state changed, this is tappable, look here. The engineering half of the story is which properties are cheap to animate — get that wrong and animations stutter on mid-range phones.

Transitions — Animating State Changes

.btn {
  background: var(--brand);
  transition: background-color .15s ease, transform .15s ease;
}
.btn:hover  { background: var(--brand-dark); transform: translateY(-1px); }
.btn:active { transform: translateY(0); }

Formula: transition: property duration timing-function [delay]. Durations that feel right: 100–200ms for hovers, 200–350ms for panels/modals. Never transition: all — it animates properties you didn't intend, including layout ones.

Keyframe Animations — Multi-Step & Looping

@keyframes pulse {
  0%, 100% { opacity: 1; }
  50%      { opacity: .35; }
}
.skeleton { animation: pulse 1.5s ease-in-out infinite; }

@keyframes slide-in {
  from { transform: translateX(100%); opacity: 0; }
  to   { transform: translateX(0);    opacity: 1; }
}
.toast { animation: slide-in .3s ease-out; }

The Performance Law: Compositor Properties Only

Animate freely (GPU, no layout)Avoid animating (forces reflow/repaint)
transform (translate/scale/rotate)width, height, top, left, margin
opacitybox-shadow (animate a pseudo-element's opacity instead)
filter (mostly)background-position, font-size
/* ❌ janky drawer */  .drawer { left: -300px; transition: left .3s; }
/* ✅ 60fps drawer */  .drawer { transform: translateX(-100%); transition: transform .3s; }

Why: transform/opacity changes run on the compositor thread — they stay smooth even while JS is busy. Layout properties re-run layout for the whole page, on the main thread, every frame.

Respecting Users (non-negotiable)

@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: .01ms !important;
    transition-duration: .01ms !important;
  }
}
💡 Production tip: diagnose jank with DevTools → Performance → record the interaction. Green = compositing (fine); purple layout blocks every frame = you're animating a layout property. Also know animation-play-state: paused — pause looping animations when the tab is hidden to save battery.
