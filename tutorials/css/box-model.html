Box Model Basics

Every element is a rectangle of four nested layers. Layout is nothing but arranging these boxes — understand this one diagram and half of CSS confusion disappears.

The Four Layers

┌─────────────── margin (transparent, pushes neighbors away) ───────────────┐
│  ┌───────────── border ─────────────────────────────────────────────┐     │
│  │  ┌─────────── padding (space inside, shows background) ─────┐    │     │
│  │  │                     content                              │    │     │
│  │  └──────────────────────────────────────────────────────────┘    │     │
│  └───────────────────────────────────────────────────────────────────┘    │
└────────────────────────────────────────────────────────────────────────────┘

The Fix Everyone Ships: border-box

/* Default (content-box): width = CONTENT only
   .card { width: 300px; padding: 20px; border: 1px }  → really 342px wide 😡 */

*, *::before, *::after { box-sizing: border-box; }
/* Now width = content + padding + border → 300px means 300px 😌 */

This reset is in effectively every production stylesheet on Earth (including this site's).

Margin's Two Famous Behaviors

/* 1. Collapsing: vertical margins of stacked blocks MERGE (larger wins) */
p { margin: 24px 0; }   /* gap between two p's = 24px, not 48px */
/* Doesn't happen inside flex/grid containers — one more reason they won */

/* 2. Centering a block: */
.container { max-width: 1100px; margin: 0 auto; }   /* the classic */

Shorthand You'll Read Every Day

margin: 8px;                  /* all four */
margin: 8px 16px;             /* vertical | horizontal */
margin: 8px 16px 24px 4px;    /* top right bottom left — clockwise */
padding-inline: 16px;         /* logical: left+right (flips for RTL languages) */

display — What Kind of Box?

ValueBehavior
blockFull width, stacks vertically (div, p, h1)
inlineFlows in text; width/height/vertical margins ignored (span, a)
inline-blockFlows in text but respects box properties
flex / gridBecomes a layout container for its children (next lessons)
noneGone — no box, no space (vs visibility: hidden which keeps the space)
💡 Production tip: when spacing looks wrong, open DevTools and hover the element — the margin (orange) / padding (green) overlay shows exactly where space comes from. Also: prefer gap (flex/grid) over margins between siblings — no collapsing, no last-child exceptions.
