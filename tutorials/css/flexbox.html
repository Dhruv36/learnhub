Flexbox Intermediate

Flexbox distributes space and aligns items along one axis. It's content-out: items declare how they'd like to grow and shrink, and the container negotiates. Toolbars, navbars, form rows, card innards, centering — flexbox. Two-dimensional page shells — Grid (next lesson). This lesson covers the full algorithm, because the algorithm is what interviews and gnarly bugs actually test.

The Mental Model: Two Axes

.container { display: flex; flex-direction: row; }        /* row (default) | column | *-reverse */

        main axis (direction of flow) ─────────────▶
      ┌──────────┬──────────┬──────────┐    ▲
      │  item 1  │  item 2  │  item 3  │    │  cross axis
      └──────────┴──────────┴──────────┘    ▼

justify-content → distributes along the MAIN axis
align-items     → aligns along the CROSS axis
(In column direction the two swap their visual meaning — the names follow the AXES, not the screen.)

Item Properties — How the Space Negotiation Works

flex:   ;

flex: 0 1 auto;    /* the DEFAULT: don't grow, may shrink, start from width/content */
flex: 1;           /* = 1 1 0    → share ALL space equally (basis 0 ignores content size!) */
flex: auto;        /* = 1 1 auto → grow from natural size — bigger content ends up bigger */
flex: none;        /* = 0 0 auto → rigid, content-sized */
flex: 0 0 240px;   /* rigid 240px sidebar — doesn't grow, doesn't shrink */

The algorithm, in one pass: ① lay items at their basis (flex-basis, else width, else content size) → ② surplus space? distribute by grow ratios → ③ deficit? shrink by shrink ratios weighted by basis (big items give up more) → ④ but never below min-width — which defaults to auto ≈ content size. That last clause is the most important sentence in this lesson:

🐛 THE flexbox bug: a long URL / unbroken string / wide table inside a flex item blows the layout open, because min-width: auto forbids shrinking below content. Fix: min-width: 0 (or an explicit width/overflow) on the item. Only then can overflow: hidden; text-overflow: ellipsis; white-space: nowrap truncate. Commit this to memory — it accounts for a comical share of all flexbox debugging hours.
Item propertyWhat it does
align-selfOverride align-items for one item (one bottom-aligned button in a top-aligned row)
orderVisual reorder (default 0, lower first) — DOM order still rules tab/screen-reader flow
margin: auto (any side)Absorbs ALL free space on that side — margin-left: auto pushes an item (and followers) to the far end. Also: auto margins fully center a lone item, and they beat justify-content

Worked Example 1 — the Navbar (annotated)

.nav {
  display: flex;
  align-items: center;      /* logo, links, avatar on one midline */
  gap: 1rem;
  padding-inline: 1.5rem;
}
.nav__links {
  display: flex; gap: 1rem;
  margin-left: auto;         /* ← the auto-margin trick: everything after this hugs the right */
}
/* Wins over 'float: right' era: vertical centering free, wrap control, gap spacing, RTL-safe */

Worked Example 2 — Media Object + Truncation

.comment { display: flex; gap: .8rem; }
.comment__avatar { flex: none; }               /* never squish the face */
.comment__body   { flex: 1; min-width: 0; }    /* fill the row; ALLOW shrinking (the fix!) */
.comment__name   { overflow: hidden; text-overflow: ellipsis; white-space: nowrap; }

Worked Example 3 — Sticky-Footer Page

body { min-height: 100dvh; display: flex; flex-direction: column; }
main { flex: 1; }        /* absorbs leftover height → footer at bottom on short pages,
                            pushed down naturally on long ones. Three lines, no hacks. */

Worked Example 4 — Equal-Height Cards with Bottom-Pinned Buttons

.cards { display: flex; gap: 1rem; }
.card  { flex: 1; display: flex; flex-direction: column; }   /* stretch (default) equalizes heights */
.card .btn { margin-top: auto; }    /* auto-margin in COLUMN direction pins the button down —
                                        every card's button on one line regardless of text length */

flex: 1 vs flex: auto — the Distinction That Signals Mastery

/* Three tabs: 'Home' 'Products' 'Contact us and support' */
.tab { flex: 1; }      /* basis 0: content ignored → all tabs EXACTLY equal width */
.tab { flex: auto; }   /* basis auto: natural sizes grow by equal ADDED amounts → longer tab stays longer */

Interview Questions

Q1: Walk through how flexbox resolves sizes when space is short.

Items start at flex-basis (falling back to width, then content). The deficit is distributed by shrink factors weighted by basis — larger items surrender more. Shrinking halts at each item's min-width, which defaults to auto (≈ min-content), so unbreakable content creates overflow unless min-width: 0 opts out. Mentioning the min-width: auto floor is what separates "uses flexbox" from "understands flexbox".

Q2: flex: 1 vs flex: auto vs flex: none — one sentence each.

flex:1 (1 1 0) divides ALL space equally ignoring content; flex:auto (1 1 auto) grows items from their natural sizes by equal amounts; flex:none (0 0 auto) is rigid at natural size. Equal tabs vs proportional tabs vs fixed chips.

Q3: How do auto margins interact with justify-content?

Auto margins absorb free space FIRST — if any item has an auto margin on the main axis, there's no free space left for justify-content to distribute, so it does nothing. That's both the navbar trick (margin-left: auto) and a debugging gotcha.

Q4: When is flexbox the wrong tool?

Two-dimensional alignment (rows AND columns must line up — grid), strict column layouts where content length must not affect widths (grid's container-driven tracks), and large visual reordering (a11y). Flex is for one-axis distribution where content size SHOULD matter.

🎯 Key Takeaways

  • Two axes: justify-content = main, align-items = cross; the names follow flex-direction, not the screen.
  • flex: grow shrink basis — basis 0 shares equally, basis auto grows from content; shrink is weighted by size.
  • min-width: auto is the hidden floor — min-width: 0 is the fix for every "flex item won't shrink/truncate" bug.
  • Auto margins eat free space before justify-content — and are the idiomatic push-right / pin-bottom tool.
  • gap over margins; wrap is opt-in; visual reordering never changes tab order.
  • Flex = 1D, content-out. Grid = 2D, container-in. They compose — grid the page, flex the innards.
