Responsive Design Intermediate

Over 60% of web traffic is mobile. Responsive isn't a feature — it's the baseline. The modern approach: intrinsically flexible layouts first, media queries only for what remains.

Prerequisite (from the HTML track)

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Mobile-First Media Queries

/* Base styles = the MOBILE layout (simplest case, no query) */
.nav-links { display: none; }

/* Layer complexity on as space appears: */
@media (min-width: 768px)  { .nav-links { display: flex; } }
@media (min-width: 1100px) { .layout { grid-template-columns: 240px 1fr; } }

Mobile-first (min-width, adding features) beats desktop-first (max-width, undoing them) because undoing styles is where bugs live. Pick breakpoints where your content breaks, not at specific devices — 768/1024/1280 are conventions, not laws.

The Modern Toolkit (queries are the last resort)

/* Fluid grids — zero queries: */
.cards { display: grid; grid-template-columns: repeat(auto-fill, minmax(220px, 1fr)); gap: 1rem; }

/* Fluid type & spacing: */
h1 { font-size: clamp(1.8rem, 4vw, 2.6rem); }

/* Fluid media: */
img, video { max-width: 100%; height: auto; }

/* Wide tables scroll in place instead of breaking the page: */
.table-wrap { overflow-x: auto; }

Queries Beyond Width

@media (prefers-color-scheme: dark) { :root { --bg: #111; --ink: #eee; } }
@media (prefers-reduced-motion: reduce) { * { animation: none !important; transition: none !important; } }
@media (hover: none) { .tooltip-on-hover { display: none; } }   /* touch screens */
@media print { .sidebar, .nav { display: none; } }

prefers-reduced-motion is an accessibility requirement — vestibular disorders make animated interfaces physically unpleasant. Respect it in every production stylesheet.

Testing Like Production Teams Do

DevTools device toolbar (Ctrl+Shift+M) for fast iteration — then at least one real phone before shipping, because URL-bar resizing, notches (env(safe-area-inset-*)), and touch targets (44px minimum) only show up on hardware.

💡 Production tip: if a component needs to adapt to its container rather than the viewport (same card in a wide main area vs a narrow sidebar), media queries can't express that — that's Container Queries, two lessons ahead.
