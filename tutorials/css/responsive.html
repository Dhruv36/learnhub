Responsive Design Intermediate

Over 60% of traffic is mobile, and Google indexes the mobile rendering of your site first. Responsive isn't a feature — it's the substrate. The modern method is a hierarchy: ① intrinsically flexible layouts → ② fluid values → ③ media queries only for what remains. Sites written query-first in 2015 style carry 3× the CSS and 10× the bugs.

Level 0 — the Prerequisite (from the HTML track)

<meta name="viewport" content="width=device-width, initial-scale=1.0">
/* Without it: phones render a 980px virtual page zoomed out; queries never fire.
   Never add user-scalable=no / maximum-scale=1 — blocking pinch-zoom fails WCAG. */

Level 1 — Intrinsic Layout (zero queries)

/* Grids that re-column themselves: */
.cards { display: grid; grid-template-columns: repeat(auto-fill, minmax(220px, 1fr)); gap: 1rem; }

/* Rows that wrap when they must: */
.toolbar { display: flex; flex-wrap: wrap; gap: .5rem 1rem; }

/* The sidebar pattern — sidebar fixed-ish, content flexible, wraps under pressure: */
.with-sidebar { display: flex; flex-wrap: wrap; gap: 1rem; }
.sidebar      { flex: 1 1 240px; }
.content      { flex: 999 1 60%; min-width: 0; }   /* huge grow: content takes the row when wrapped */

/* Media that never overflows: */
img, video, canvas { max-width: 100%; height: auto; }

/* Wide data that scrolls IN PLACE (the page itself must never scroll sideways): */
.table-wrap { overflow-x: auto; }

Level 2 — Fluid Values

h1       { font-size: clamp(1.8rem, 1.2rem + 2.5vw, 2.6rem); }   /* rem term keeps zoom legal */
.section { padding-block: clamp(2rem, 6vw, 5rem); }
.wrap    { width: min(100% - 2rem, 1100px); margin-inline: auto; }  /* container + gutters, one line */

Level 3 — Media Queries (for what remains: page-level shifts)

/* MOBILE-FIRST: base styles are the phone layout; min-width queries ADD as space appears */
.nav-links { display: none; }                     /* base: burger menu */
@media (min-width: 48rem)  { .nav-links { display: flex; } .burger { display: none; } }
@media (min-width: 68.75rem) { .layout { grid-template-columns: 240px 1fr; } }

/* Modern range syntax reads like math: */
@media (48rem <= width < 68.75rem) { … }          /* tablet band, no boundary double-fire */
PracticeWhy
Mobile-first (min-width)Adding complexity beats undoing it — desktop-first max-width sheets accumulate override bugs
Breakpoints in rem, not pxA user with 20px base font hits the "tablet" layout when the CONTENT needs it, not at an arbitrary px count
Breakpoints where YOUR content breaksResize until it looks wrong; that's the breakpoint. 768/1024 are folklore, not law
Few breakpoints (2–3)If you need six, levels 1–2 are underused

Queries Beyond Width — the Preference & Capability Set

@media (prefers-color-scheme: dark)     { :root { --bg:#0d1117; --ink:#e6edf3; } }
@media (prefers-reduced-motion: reduce) { *,*::before,*::after {
                                            animation-duration:.01ms !important;
                                            transition-duration:.01ms !important; } }
@media (prefers-contrast: more)         { :root { --border: #000; } }
@media (hover: none)                    { .reveal-on-hover { opacity: 1; } }  /* touch: always visible */
@media (pointer: coarse)                { .btn { min-height: 44px; } }         /* fat-finger targets */
@media (forced-colors: active)          { .badge { border: 1px solid; } }     /* High Contrast Mode */
@media print                             { .sidebar, nav, .ads { display: none; }
                                           a[href^="http"]::after { content: " (" attr(href) ")"; } }

prefers-reduced-motion is an accessibility requirement (vestibular disorders); hover: none/pointer: coarse detect input capability rather than guessing from screen size — a phone-sized window on a desktop still has hover.

Worked Example — a Product Header, All Levels Together

.header {
  display: flex; flex-wrap: wrap; align-items: center;
  gap: .75rem 1.5rem;                                /* L1: wraps gracefully */
  padding-inline: clamp(1rem, 4vw, 2.5rem);          /* L2: fluid gutters */
  padding-top: env(safe-area-inset-top);             /* hardware */
}
.search { flex: 1 1 200px; min-width: 0; }           /* L1: grows, shrinks, truncates */
@media (width < 36rem) { .search { order: 1; flex-basis: 100%; } }   /* L3: one true breakpoint —
                                                        search drops to its own row on phones */

Interview Questions

Q1: Why mobile-first, concretely — not as a slogan?

Base styles serve the simplest layout, so the weakest devices parse the least CSS and get the best-tested path; min-width queries ADD structure, which composes, while max-width queries UNDO styles, which conflicts; and it forces content priority decisions early. Bonus: Google's mobile-first indexing judges the mobile experience as THE experience.

Q2: When is a media query the wrong tool even though it works?

When the component should respond to its CONTAINER, not the viewport (same card in sidebar vs main — container queries, next lessons); when a fluid value suffices (clamp beats three font-size breakpoints); when wrapping is the natural behavior (flex-wrap beats stacking queries).

Q3: How do you hunt a horizontal-scroll bug on mobile?

DevTools: * { outline: 1px solid red } (outline doesn't shift layout) or iterate document.querySelectorAll('*') checking el.scrollWidth > document.documentElement.clientWidth. Usual suspects: unwrapped tables/pre, 100vw on a page WITH a scrollbar, absolutely positioned decorations, negative margins. Then fix the element, don't slap overflow-x:hidden on body (that also kills sticky).

Q4: Which user-preference queries does a production stylesheet honor, minimum?

prefers-reduced-motion (a11y requirement), prefers-color-scheme (paired with a manual override), and forced-colors (High Contrast survival). prefers-contrast and print round out the set. These are cheap to support and audits check them.

🏋️ Practice (Playground):
1. Build the sidebar pattern (flex: 1 1 240px / flex: 999 1 60%) and resize — find the exact width where it wraps, with zero queries.
2. Take a 3-breakpoint font-size setup and replace it with one clamp(); verify zoom still scales it.
3. Add the reduced-motion kill-switch and toggle "Emulate prefers-reduced-motion" in DevTools Rendering panel.
4. Create a deliberate horizontal overflow (wide pre), then find it with the red-outline trick and fix it properly.
5. Write the print query for an article page and check Print Preview.

🎯 Key Takeaways

  • Hierarchy: intrinsic layout → fluid values → few, content-driven, rem-based, mobile-first queries.
  • auto-fill grids, flex-wrap patterns, min(100% - 2rem, 1100px) containers do most responsive work free.
  • Preference queries (reduced-motion, color-scheme, forced-colors) and capability queries (hover, pointer) are part of "responsive".
  • Mobile hardware realities: dvh, safe-area insets, 16px inputs, 44px targets, scrollbar-gutter.
  • The page never scrolls sideways — wide content scrolls inside its own wrapper.
  • Emulators for iteration, real hardware before shipping.
