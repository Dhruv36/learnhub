Generators & Iterators Advanced

The protocol behind for…of, spread, and destructuring — and the tool for lazy sequences that don't fit in memory: paginated APIs, log streams, infinite series.

The Iterator Protocol (what for…of actually calls)

const it = [10, 20][Symbol.iterator]();
it.next();   // { value: 10, done: false }
it.next();   // { value: 20, done: false }
it.next();   // { value: undefined, done: true }

// Anything implementing this works with for…of, [...spread], destructuring:
// Arrays, strings, Map, Set, NodeList — and your own objects.

Generators — Iterators Without the Boilerplate

function* idGenerator() {              // the * makes it a generator
  let id = 1;
  while (true) yield id++;             // yield = pause here, hand out a value
}
const ids = idGenerator();
ids.next().value;   // 1
ids.next().value;   // 2 — state (id) survives between calls, like a resumable function

function* take(n, iterable) {          // generators compose
  for (const x of iterable) {
    if (n-- <= 0) return;
    yield x;
  }
}
[...take(3, idGenerator())];           // [1, 2, 3] — laziness makes "infinite" safe

The Production Use Case: Async Generators

async function* fetchAllUsers() {              // paginated API → one clean stream
  let url = "/api/users?page=1";
  while (url) {
    const res = await fetch(url);
    const { items, nextPage } = await res.json();
    yield* items;                               // yield* = delegate each item
    url = nextPage;
  }
}

for await (const user of fetchAllUsers()) {     // consumer neither knows nor cares
  if (user.flagged) await review(user);         // about pages — and can stop early:
  if (enough) break;                            // remaining pages are NEVER fetched
}

This pattern — pagination, streaming uploads, tailing logs, Node streams (which are async-iterable) — is where generators earn their place in real codebases.

Making Your Own Objects Iterable

class TimeRange {
  constructor(start, end) { this.start = start; this.end = end; }
  *[Symbol.iterator]() {
    for (let d = new Date(this.start); d <= this.end; d.setDate(d.getDate() + 1))
      yield new Date(d);
  }
}
[...new TimeRange("2026-01-01", "2026-01-03")];   // three Date objects
💡 Interview context: generators historically powered async flows (co, Redux-Saga) before async/await standardized the idea — async/await is essentially generators + a promise-driving loop. Also know gen.return()/gen.throw(): consumers can inject early termination or errors into the paused function — that's how break in for await triggers cleanup upstream.
