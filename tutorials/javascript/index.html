Variables, Types & Operators Basics

JavaScript runs everywhere — every browser, every Node server, most interview screens. It's also full of historical quirks; this track teaches the modern subset production teams actually use, and flags the traps by name.

Declaring Variables (the 2-rule system)

const apiUrl = "https://api.app.com";   // default: cannot be reassigned
let retries = 0;                         // only when reassignment is NEEDED
retries += 1;

// var is legacy — function-scoped, hoisted, allows redeclaration. Don't write new var.
// Note: const prevents REASSIGNMENT, not mutation — const arr = []; arr.push(1) is fine.

The Types

typeof "hi"        // "string"     typeof 42          // "number" (only one number type)
typeof true        // "boolean"    typeof undefined   // "undefined"
typeof 10n         // "bigint"     typeof Symbol()    // "symbol"
typeof {}          // "object"     typeof []          // "object"  ← arrays are objects
typeof null        // "object"  ← 30-year-old bug, kept for compatibility
typeof function(){} // "function"

undefined = never assigned; null = deliberately empty. Numbers are IEEE-754 floats: 0.1 + 0.2 === 0.30000000000000004 — money is stored in integer cents, never floating dollars.

Equality — Always ===

1 == "1"     // true  — == coerces types with rules nobody fully remembers
1 === "1"    // false — === compares type AND value. Use it always.
null == undefined   // true (the ONE handy == case: x == null checks both)
NaN === NaN  // false! use Number.isNaN(x)

Truthiness & the Modern Operators

// Falsy values — memorize all eight:
false, 0, -0, 0n, "", null, undefined, NaN      // EVERYTHING else is truthy ("0" is truthy!)

const name = user.nickname ?? "Anonymous";      // ?? — default only for null/undefined
const city = user?.address?.city;               // ?.  — safe navigation, no TypeError
const label = count > 1 ? "items" : "item";     // ternary
// Trap: `count || 10` replaces a legitimate 0 with 10 — that's why ?? exists.

Template Literals

const msg = `Hi ${user.name}, you have ${count} new ${label}.`;   // interpolation
const sql = `SELECT *
  FROM users`;                                                     // multi-line works too
💡 Production tip: teams enforce these choices mechanically — ESLint rules no-var, prefer-const, eqeqeq ship in every real config. You'll meet the tooling in the Modules lesson; the point is: style laws are automated, never argued in review.
