DOM & Events Basics

The DOM is the live object tree the browser built from your HTML; events are how it talks back. Frameworks abstract this — but framework bugs are debugged at this level, and interviews check it directly.

Selecting & Changing

const form   = document.querySelector("#signup");        // first match (any CSS selector)
const inputs = document.querySelectorAll("input");        // static NodeList of all

el.textContent = user.bio;            // ✅ safe text (XSS-proof)
el.innerHTML   = "<b>hi</b>";         // ⚠ parses HTML — never with user data (HTML track: Security)
el.classList.add("active");           // add / remove / toggle / contains
el.dataset.userId = "42";             // ↔ data-user-id attribute
el.setAttribute("aria-expanded", "true");

Creating & Inserting

const li = document.createElement("li");
li.textContent = task.title;
list.append(li);                       // also: prepend, before, after, el.remove()

// Batch inserts touch the DOM once (each touch can trigger style/layout work):
const frag = document.createDocumentFragment();
tasks.forEach(t => { const li = document.createElement("li"); li.textContent = t.title; frag.append(li); });
list.append(frag);

Events — Listen, Read, Prevent

button.addEventListener("click", (e) => {
  e.preventDefault();                  // stop default action (link nav, form submit)
  console.log(e.target);               // the element actually clicked
});

form.addEventListener("submit", (e) => {
  e.preventDefault();                  // SPA-style: handle it in JS instead of page reload
  const data = Object.fromEntries(new FormData(form));   // {email: "...", plan: "pro"}
});

input.addEventListener("input", e => search(e.target.value));   // fires per keystroke
// key events: e.key === "Escape" | "Enter" | "ArrowDown" …

Bubbling & Delegation — the Senior Pattern

// Events BUBBLE: child → parent → … → document. So one listener can serve 1000 rows:
table.addEventListener("click", (e) => {
  const btn = e.target.closest("[data-action='delete']");   // walk up from the click
  if (btn && table.contains(btn)) deleteRow(btn.closest("tr").dataset.id);
});

Delegation = fewer listeners, and rows added later work automatically. It's how frameworks implement their event systems internally. e.stopPropagation() halts bubbling — use sparingly; it breaks analytics and outside-click handlers silently.

Cleanup — Listeners Are a Contract

const controller = new AbortController();
window.addEventListener("resize", onResize, { signal: controller.signal });
controller.abort();                    // removes every listener tied to the signal
// Also: { once: true } for self-removing listeners.
// Forgotten listeners on removed DOM = memory leaks (Ultra-Advanced: Memory lesson).
💡 Production tip: scripts loaded with defer run after the DOM is parsed — the old DOMContentLoaded wrapper is unnecessary. And before reaching for any DOM library: this entire lesson is the "vanilla JS" that renders this site.
