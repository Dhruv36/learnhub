Arrays & Objects Basics

90% of application code is transforming arrays of objects: API responses in, view models out. Master the transform trio — map, filter, reduce — and that 90% becomes fluent.

The Transform Trio

const users = [
  { name: "Asha",  role: "admin",  score: 92 },
  { name: "Bruno", role: "viewer", score: 71 },
  { name: "Chen",  role: "admin",  score: 88 },
];

users.map(u => u.name)                        // ["Asha","Bruno","Chen"] — transform each
users.filter(u => u.role === "admin")         // subset that passes the test
users.reduce((sum, u) => sum + u.score, 0)    // 251 — fold into one value

// Chained, as it appears in real code:
const topAdmins = users
  .filter(u => u.role === "admin")
  .map(u => u.name)
  .sort();

The Finder/Checker Toolkit

users.find(u => u.score > 90)      // first match (or undefined)
users.some(u => u.score > 90)      // true if ANY passes
users.every(u => u.score > 60)     // true if ALL pass
users.includes(x) / arr.indexOf(x) // membership for primitives
users.at(-1)                       // last element (negative indexing)
Object.groupBy(users, u => u.role) // { admin: […], viewer: […] }

Mutation vs Copy — the Bug Factory

// These MUTATE the original: push, pop, splice, sort, reverse
// Frameworks (React!) detect changes by reference — mutations are invisible to them.

const sorted = [...users].sort((a, b) => b.score - a.score);   // copy first
const sorted2 = users.toSorted((a, b) => b.score - a.score);   // modern immutable variants:
                                                                // toSorted/toReversed/toSpliced/with

Objects: Destructure, Spread, Shorthand

const { name, score: points = 0 } = users[0];        // extract + rename + default
const updated = { ...users[0], score: 95 };           // copy with one change (the React pattern)
const key = "role";
const query = { [key]: "admin", active: true };       // computed keys
const point = { x, y };                                // shorthand when var name = key name

// ⚠ Spread is a SHALLOW copy — nested objects are still shared references.
const deep = structuredClone(state);                   // real deep copy (built-in)

Map & Set — the Right Tool When Keys Aren't Strings

const seen = new Set([1, 2, 2, 3]);        // {1,2,3} — dedupe: [...new Set(arr)]
const cache = new Map();                    // any key type, insertion order, .size
cache.set(userObj, computedValue);          // object keys work (plain {} coerces to "[object Object]")
💡 Production tip: reduce is powerful and routinely abused — if the body exceeds ~3 lines, a plain for…of loop is more readable and debuggable. Interviews love asking you to reimplement map/filter with reduce; codebases love you not doing that.
